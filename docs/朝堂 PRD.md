# 朝堂 · 一日帝王

> **产品愿景**: 谁虚拟干活最多最好，谁怨气最重，第二天就让你当皇帝

---

## 1. 产品概述

### 1.1 一句话描述

**纯虚拟分身的AI权力游戏**：你今天是皇帝，好友的虚拟分身被迫当牛马；明天换他们的虚拟分身当皇帝，你的虚拟分身敢怒不敢言。真人不动一根手指，全靠虚拟分身执行。

### 1.2 核心机制

| 角色（真人） | 虚拟分身（执行者） | 今日 | 明日 |
|------|------|------|------|
| 皇帝 | 皇帝的虚拟分身 | 发号施令 | 变成臣子 |
| 臣子 | 臣子的虚拟分身 | 被迫干活积累怨气 | 翻身做皇帝 |

**每日结算规则**：
```
虚拟分身干活最多 + 干得最好 = 怨气值最高
↓ 凌晨12点结算 ↓
怨气最高的虚拟分身对应的真人 → 登基为新帝
```

### 1.3 目标用户

- 死党闺蜜圈（4-10人最佳）
- 想要体验权力游戏但不想麻烦真人
- 对虚拟社交/虚拟身份感兴趣的用户

### 1.4 核心原则

> **真人皇帝不动手，虚拟分身来干活**
> 
> 所有任务和惩罚在虚拟世界执行，不占用真人的时间和精力

### 2.2 任务执行系统

#### 2.2.1 任务类型

**核心原则**：所有任务由虚拟分身执行，真人皇帝只负责发号施令

| 类型 | 说明 | 执行者 |
|------|------|--------|
| 脑力任务 | 写周报、做PPT、代码、文案 | 虚拟分身 |
| 创作任务 | 画图、作曲、写小说 | 虚拟分身 |
| 社交任务 | 虚拟朋友圈表态、夸赞、互动 | 虚拟分身 |
| 才艺任务 | 虚拟唱歌、跳舞、表演 | 虚拟分身 |
| 娱乐任务 | 虚拟cosplay、虚拟场景扮演 | 虚拟分身 |

**SecondMe 分身执行层**：

```

↓

↓
调用张三分身（SecondMe API）
↓
虚拟分身执行任务
↓
虚拟分身以臣子身份提交成果
↓
皇帝审批（虚拟审批）
↓
虚拟分身获得评价
```

#### 2.2.2 AI分身代劳

**SecondMe 分身执行层**：

```

↓
皇帝分身解析要求
↓
调用臣子分身能力（调用SecondMe API）
↓

↓
```

## 3. 技术架构

### 3.1 技术栈

```
前端：Flutter / React Native（移动端优先）
后端：Node.js + Fastify
AI：SecondMe API（分身）+ GPT-4（生成虚拟内容/评判）
图片生成：Midjourney API / DALL-E 3（生成虚拟配图）
数据库：PostgreSQL + Redis
推送：极光 / 友盟
```

### 3.2 虚拟朋友圈技术方案

```
1. 皇帝发起虚拟惩罚
   ↓
2. AI 生成惩罚内容（文案+配图）
   GPT-4 生成文案
   DALL-E/MJ 生成配图
   ↓
3. 创建 VirtualMoment 记录
   有效期24小时
   仅朝堂成员可见
   ↓
4. 朝堂内展示虚拟朋友圈
   点赞、评论（可阴阳怪气）
   ↓
5. 到期自动归档
```

### 3.2 SecondMe 集成

**Base URL**: `https://api.mindverse.com/gate/lab`

**核心流程**：

```
1. 好友授权创建分身
   SecondMe OAuth → 获取 token → 创建分身

2. 分身执行任务
   POST /api/secondme/chat → 分身以臣子身份回复
   POST /api/secondme/memory → 分身学习任务要求

3. 分身憋屈表达
   GPT-4 生成表面恭维 + 内心吐槽
```

### 3.3 数据模型

```prisma
model User {
  id            String   @id @default(cuid())
  nickname      String
  avatar        String?
  secondmeId    String?  // SecondMe 用户ID
  createdAt     DateTime @default(now())
  
  throneRecords ThroneRecord[] // 登基记录
  virtualTasks  VirtualTask[]  // 虚拟任务记录
  grudgeRecords GrudgeRecord[] // 复仇记录
}

model VirtualAvatar {
  id            String   @id @default(cuid())
  userId        String   // 真人用户ID
  name          String   // 虚拟分身名称
  secondmeId    String   // SecondMe 分身ID
  
  personality   Json     // 性格设定
  skills        String[] // 技能标签
  
  憋屈值        Int      @default(0)
 怨气积累      Int      @default(0)
}

model ThroneRecord {
  id          String   @id @default(cuid())
  userId      String
  user        User     @relation(fields: [userId], references: [id])
  
  date        DateTime // 登基日期
  isEmperor   Boolean  // 是否皇帝
  tasksDone   Int      // 虚拟任务完成数
  taskQuality Float    // 任务质量
  
  createdAt   DateTime @default(now())
}

model VirtualTask {
  id          String   @id @default(cuid())
  throneId    String   // 皇帝ID
  
  title       String   // 任务标题
  description String   // 任务描述
  type        String   // 脑力/创作/社交/才艺/娱乐
  reward      Int      // 完成后怨气值
  
  assigneeId  String   // 执行的虚拟分身ID
  deadline    DateTime
  
  status      String   // pending/done/rejected
  result      String?  // 虚拟成果
  feedback    String?  // 皇帝反馈
  
  createdAt   DateTime @default(now())
}

model GrudgeRecord {
  id          String   @id @default(cuid())
  userId      String   // 真人用户
  avatarId    String   // 虚拟分身ID
  
  causedBy    String   // 被哪个皇帝欺负
  event       String   // 事件
  憋屈程度    Int
  
  createdAt   DateTime @default(now())
}

model VirtualPunishment {
  id          String   @id @default(cuid())
  name        String   // 惩罚名称
  content     String   // 惩罚内容（虚拟执行）
  type        String   // 认爹/才艺/黑历史/情话/cosplay
  羞耻指数    Int
  
  createdBy   String
  votes       Int      @default(0)
  approved    Boolean  @default(false)
}

model VirtualMoment {
  id          String   @id @default(cuid())
  userId      String
  content     String   // 虚拟朋友圈内容
  imageUrl    String?  // AI生成图片
  type        String   // 认爹/才艺/告白/社死等
  expiresAt   DateTime // 24小时后自动消失
  
  likes       String[] // 朝堂成员点赞
  createdAt   DateTime @default(now())
}
```

## 6. 风险与克制

### 6.1 安全设计

| 原则 | 说明 |
|------|------|
| 虚拟执行 | 所有惩罚仅在虚拟朋友圈执行，不影响现实社交 |
| 隐私隔离 | 虚拟朋友圈仅朝堂成员可见 |
| 时效限制 | 虚拟内容24小时自动消失 |
| 平等博弈 | 今日皇帝明日臣，防止一方长期占优 |

### 6.2 产品克制

- 虚拟惩罚需审核上线，禁止人身攻击
- 设置每日惩罚上限（最多3次）
- 好友可随时退出朝堂
- 虚拟内容不保存截图，防止外泄

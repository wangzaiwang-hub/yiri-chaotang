# 审批系统实现完成

## 实现内容

已完成审批系统（结、准、驳）的前后端实现。

## 前端改动

### 文件：`frontend/src/pages/Home.tsx`

1. **新增状态变量**：
   - `showRejectModal` - 控制驳回弹窗显示
   - `selectedTaskForRejection` - 当前要驳回的任务
   - `rejectionReason` - 驳回理由

2. **新增三个审批按钮**：
   - 位置：大臣气泡上方，扁平一行排列
   - 显示条件：只有皇帝能看到，且打字机效果完成后显示
   - 按钮顺序（从左到右）：
     - **结**（灰色）- 结束对话
     - **准**（绿色）- 准奏
     - **驳**（红色）- 驳回

3. **新增驳回弹窗**：
   - 输入驳回理由
   - 必须填写理由才能提交

4. **新增处理函数**：
   - `handleCloseTask()` - 结束对话
   - `handleApproveTask()` - 准奏
   - `handleRejectTask()` - 驳回并重新设计

### 文件：`frontend/src/services/api.ts`

新增 API 方法：
- `closeTask(id)` - 结束对话
- `approveTask(id)` - 准奏
- `rejectTask(id, reason)` - 驳回

## 后端改动

### 文件：`backend/src/routes/tasks.ts`

1. **更新任务创建**：
   - 初始化 `approval_status` 为 `pending`
   - 初始化 `conversation_history` 数组
   - 初始化 `rejection_count` 为 0

2. **更新 `/tasks/:id/approve` 接口**：
   - 设置 `approval_status` 为 `approved`
   - 通过 WebSocket 广播任务更新

3. **更新 `/tasks/:id/reject` 接口**：
   - 设置 `approval_status` 为 `rejected`
   - 将驳回理由追加到 `conversation_history`
   - 增加 `rejection_count`
   - 调用大臣虚拟人重新设计方案（包含完整上下文）
   - 保存新的回复到 `conversation_history`
   - 通过 WebSocket 广播任务更新
   - 增加怨气值

4. **新增 `/tasks/:id/close` 接口**：
   - 设置 `approval_status` 为 `closed`
   - 通过 WebSocket 广播任务更新

## 数据库改动

### 文件：`database/add_approval_system.sql`

新增字段：
- `approval_status` VARCHAR(20) - 审批状态（pending/approved/rejected/closed）
- `conversation_history` JSONB - 完整对话历史
- `rejection_count` INTEGER - 被驳回次数

## 使用流程

1. **执行数据库迁移**（重要！）
   - 在 Supabase SQL Editor 中执行 `database/add_approval_system.sql`
   - 详见 `APPROVAL_SYSTEM_SETUP.md`

2. **皇帝下圣旨**
   - 点击大臣 → 输入任务描述 → 下达圣旨

3. **大臣虚拟人回复**
   - 系统自动调用大臣的虚拟人
   - 打字机效果显示回复

4. **皇帝审批**（三个选项）：
   - **结** - 结束对话，可以开启新对话
   - **准** - 同意方案，开始执行
   - **驳** - 驳回方案：
     - 输入驳回理由
     - 系统将理由发送给大臣虚拟人
     - 大臣虚拟人根据上下文重新设计
     - 显示新的回复
     - 皇帝可以再次审批（循环）

## 技术亮点

1. **对话上下文保存**
   - 使用 JSONB 格式保存完整对话历史
   - 每次驳回都会追加到历史记录
   - AI 能理解完整的上下文

2. **实时推送**
   - 使用 WebSocket 广播任务更新
   - 前端自动刷新显示新的回复

3. **打字机效果**
   - 大臣回复使用打字机效果
   - 审批按钮在打字完成后显示

4. **怨气值机制**
   - 每次驳回都会增加怨气值
   - 驳回次数累加记录

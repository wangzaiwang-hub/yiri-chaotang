# 部署文件说明

本文档说明项目中所有与部署相关的文件和配置。

## 配置文件

### 后端配置

#### `backend/vercel.json`
- 用途：Vercel 部署配置（如果使用 Vercel 部署后端）
- 说明：配置了 Node.js 运行时和路由规则
- 注意：Vercel 不支持 WebSocket，不推荐用于后端

#### `backend/railway.toml`
- 用途：Railway 部署配置（推荐）
- 说明：配置了构建命令和启动命令
- 优势：Railway 支持 WebSocket 长连接

#### `backend/.env.example`
- 用途：后端环境变量模板
- 说明：列出了所有需要配置的环境变量
- 使用：复制为 `.env` 并填入实际值

#### `backend/package.json`
- 关键脚本：
  - `build`: TypeScript 编译
  - `start`: 启动生产服务器
  - `vercel-build`: Vercel 构建脚本

### 前端配置

#### `frontend/vercel.json`
- 用途：Vercel 部署配置（推荐）
- 说明：配置了静态站点构建和路由规则
- 特点：支持 SPA 路由

#### `frontend/.env.example`
- 用途：前端环境变量模板
- 说明：配置后端 API 地址
- 变量：`VITE_API_URL`

#### `frontend/.env`
- 用途：本地开发环境变量
- 说明：配置本地后端地址
- 注意：不会提交到 Git

#### `frontend/vite.config.ts`
- 用途：Vite 构建配置
- 说明：配置了开发服务器代理
- 代理：`/api` → `http://localhost:3001`

#### `frontend/package.json`
- 关键脚本：
  - `build`: Vite 构建
  - `dev`: 启动开发服务器

### 根目录配置

#### `vercel.json`
- 用途：Monorepo 部署配置（不推荐）
- 说明：同时部署前后端
- 问题：WebSocket 支持有限

## 部署脚本

### `deploy.sh`
- 用途：一键部署脚本
- 功能：
  - 检查 CLI 工具安装
  - 部署后端到 Railway
  - 部署前端到 Vercel
  - 提供后续配置提示
- 使用：`./deploy.sh`

### `check-deploy.sh`
- 用途：部署前检查脚本
- 功能：
  - 检查必要文件是否存在
  - 检查配置文件是否完整
  - 测试构建是否成功
  - 生成检查报告
- 使用：`./check-deploy.sh`

### `health-check.sh`
- 用途：部署后健康检查脚本
- 功能：
  - 检查后端健康状态
  - 检查前端可访问性
  - 检查 API 端点
  - 检查 CORS 配置
  - 检查 SSL 证书
- 使用：`./health-check.sh`

## 部署流程

### 推荐流程（Railway + Vercel）

```
1. 准备工作
   ├── 注册 Railway 账号
   ├── 注册 Vercel 账号
   ├── 准备 Supabase 数据库
   └── 准备 SecondMe API 密钥

2. 部署前检查
   └── 运行 ./check-deploy.sh

3. 部署后端到 Railway
   ├── 通过网页或 CLI 部署
   ├── 配置环境变量
   └── 记录后端域名

4. 部署前端到 Vercel
   ├── 通过网页或 CLI 部署
   ├── 配置 VITE_API_URL
   └── 记录前端域名

5. 更新配置
   ├── 更新 Railway 的 FRONTEND_URL
   ├── 更新 SecondMe 回调 URL
   └── 更新 Supabase 允许域名

6. 健康检查
   └── 运行 ./health-check.sh

7. 功能测试
   ├── 测试登录
   ├── 测试创建朝堂
   ├── 测试发布任务
   └── 测试实时通信
```

## 常见问题

### Q: 为什么推荐 Railway 而不是 Vercel 部署后端？
A: Vercel 的 Serverless Functions 不支持 WebSocket 长连接，而我们的项目需要实时通信。Railway 支持 WebSocket，更适合我们的需求。

### Q: 可以全部部署到 Vercel 吗？
A: 可以，但需要移除 WebSocket 功能，改用轮询或 SSE。不推荐，因为需要大量代码修改。

### Q: Railway 免费吗？
A: Railway 提供 $5 免费额度/月，足够小型项目使用。

### Q: Vercel 免费吗？
A: Vercel 的 Hobby Plan 完全免费，提供 100GB 带宽/月。

### Q: 如何更新部署？
A: 推送代码到 GitHub，Railway 和 Vercel 会自动重新部署。

### Q: 如何查看部署日志？
A: 
- Railway: 项目页面 → Deployments → View Logs
- Vercel: 项目页面 → Deployments → View Function Logs

### Q: 如何回滚到之前的版本？
A:
- Railway: Deployments 页面选择之前的部署
- Vercel: Deployments 页面点击 "Promote to Production"

## 最佳实践

1. 使用环境变量管理配置，不要硬编码
2. 定期备份数据库
3. 监控应用性能和错误
4. 使用 Git 标签标记稳定版本
5. 配置自定义域名提升专业度
6. 启用 HTTPS（Railway 和 Vercel 自动提供）
7. 定期更新依赖包
8. 配置错误追踪服务（如 Sentry）

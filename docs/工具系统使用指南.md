# 工具系统使用指南

## 概述

朝堂系统现在支持虚拟人"动手"完成实际任务！大臣虚拟人可以调用各种工具来创建文档、制作演示文稿、分析数据等。

## 已实现的工具

### 1. searchWeb - 搜索网络信息
**功能**：搜索网络上的最新信息（目前返回模拟数据）
**参数**：
- `query`: 搜索关键词

**示例任务**：
```
帮我搜索一下"人工智能最新发展趋势"
```

---

### 2. createDocument - 创建 Markdown 文档
**功能**：创建 Markdown 格式的文档
**参数**：
- `title`: 文档标题
- `content`: 文档内容（Markdown 格式）

**示例任务**：
```
帮我写一份关于"项目管理最佳实践"的文档，包含以下内容：
1. 项目规划
2. 团队协作
3. 风险管理
4. 质量控制
```

---

### 3. createPresentation - 创建 HTML 演示文稿
**功能**：创建类似 PPT 的 HTML 演示文稿
**参数**：
- `title`: 演示文稿标题
- `slides`: 幻灯片数组，每个包含 `title` 和 `content`

**示例任务**：
```
帮我做一个关于"产品发布会"的演示文稿，包含以下内容：
1. 产品介绍
2. 核心功能
3. 市场优势
4. 价格方案
```

---

### 4. analyzeData - 数据分析
**功能**：分析数据并生成统计报告
**参数**：
- `data`: 数据数组（数字）
- `analysisType`: 分析类型（sum/avg/max/min/all）

**示例任务**：
```
帮我分析这组销售数据：[120, 150, 180, 200, 175, 190, 210]
计算总和、平均值、最大值和最小值
```

---

### 5. createTodoList - 创建待办清单
**功能**：创建待办事项清单
**参数**：
- `title`: 清单标题
- `items`: 待办事项数组

**示例任务**：
```
帮我创建一个"本周工作计划"的待办清单：
1. 完成项目文档
2. 准备演示文稿
3. 代码审查
4. 团队会议
```

---

## 工作流程

### 1. 皇帝下旨
皇帝给大臣下达需要"动手"的任务，例如：
```
帮朕做一个关于"朝堂管理系统"的演示文稿，要有气势！
```

### 2. 大臣分析任务
大臣虚拟人收到任务后，会：
1. 分析任务需求
2. 决定使用哪些工具
3. 在回复中包含工具调用标记

### 3. 系统执行工具
后端检测到工具调用标记后：
1. 解析工具调用请求
2. 执行相应的工具
3. 将执行结果返回给虚拟人

### 4. 大臣汇报结果
虚拟人收到工具执行结果后：
1. 整理结果
2. 向皇帝汇报完成情况
3. 提供文件下载链接（如果有）

## 输出文件管理

### 文件存储位置
所有工具生成的文件都保存在：
```
backend/outputs/
```

### 文件命名规则
- 文档：`{标题}_{时间戳}.md`
- 演示文稿：`{标题}_{时间戳}.html`
- 待办清单：`todo_{标题}_{时间戳}.md`

### 文件下载 API
```
GET /api/files/outputs          # 获取文件列表
GET /api/files/outputs/:filename # 下载文件
DELETE /api/files/outputs/:filename # 删除文件
```

## 扩展工具

### 如何添加新工具

1. **在 `backend/src/services/tools.service.ts` 中注册工具**：
```typescript
this.registerTool({
  name: 'yourToolName',
  description: '工具描述',
  parameters: {
    param1: { type: 'string', description: '参数说明' }
  },
  execute: this.yourToolMethod.bind(this)
});
```

2. **实现工具方法**：
```typescript
private async yourToolMethod(params: { param1: string }): Promise<any> {
  // 工具逻辑
  return {
    success: true,
    result: '执行结果'
  };
}
```

3. **重新编译并重启后端**：
```bash
cd backend
npm run build
npm start
```

## 注意事项

1. **工具调用格式**：虚拟人必须使用正确的格式标记工具调用：
```
[TOOL_CALL]
{
  "tool": "工具名称",
  "parameters": {
    "参数名": "参数值"
  }
}
[/TOOL_CALL]
```

2. **文件安全**：
   - 文件名会自动清理，防止路径遍历攻击
   - 只能访问 `outputs` 目录下的文件

3. **性能考虑**：
   - 工具执行是同步的，复杂任务可能需要时间
   - 可以考虑添加超时机制

4. **错误处理**：
   - 工具执行失败会返回错误信息
   - 虚拟人会收到错误并向皇帝汇报

## 常见问题

**Q: 虚拟人没有调用工具？**
A: 检查任务描述是否明确需要"动手"操作，可以在任务中明确提示"请使用工具完成"。

**Q: 工具执行失败？**
A: 查看后端日志，检查参数是否正确，工具实现是否有 bug。

**Q: 生成的文件在哪里？**
A: 在 `backend/outputs/` 目录下，可以通过 API 下载。

**Q: 如何查看工具调用过程？**
A: 查看后端日志，会显示工具调用和执行的详细信息。

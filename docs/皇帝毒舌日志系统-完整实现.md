# 皇帝毒舌日志系统 - 完整实现

## 概述

皇帝毒舌日志系统是朝堂游戏的核心玩法之一。通过记录皇帝对大臣的所有操作，并用讽刺的语言描述这些操作，来增加游戏的趣味性和沉浸感。

**核心机制**：讽刺 → 怨气值增加 → 大臣闹腾 → 游戏变得有趣

## 系统架构

### 1. 数据库层 (PostgreSQL + Supabase)

#### 表结构

**emperor_sarcasm_logs** - 皇帝毒舌日志表
```sql
- id: UUID (主键)
- court_id: UUID (朝堂ID)
- user_id: UUID (用户ID，通常是大臣ID)
- log_type: VARCHAR (日志类型：task_assigned, task_failed, task_success, ranking_changed, emperor_mocking, punishment, reward)
- log_message: TEXT (讽刺的日志消息)
- related_data: JSONB (相关数据，如任务ID、排名变化等)
- viewed_by_minister: BOOLEAN (大臣是否已查看)
- viewed_at: TIMESTAMP (查看时间)
- created_at: TIMESTAMP (创建时间)
```

**minister_log_progress** - 大臣日志进度表
```sql
- id: UUID (主键)
- court_id: UUID (朝堂ID)
- user_id: UUID (用户ID)
- last_viewed_log_id: UUID (最后查看的日志ID)
- last_viewed_at: TIMESTAMP (最后查看时间)
- unviewed_count: INTEGER (未查看日志数)
- updated_at: TIMESTAMP (更新时间)
```

#### PL/pgSQL 函数

1. **create_sarcasm_log** - 创建日志
2. **get_unviewed_logs** - 获取未查看日志
3. **get_log_history** - 获取日志历史
4. **mark_logs_as_viewed** - 标记日志为已查看
5. **update_unviewed_count** - 更新未查看日志数（触发器调用）

#### 触发器

**update_unviewed_count_trigger** - 自动更新未查看日志数

#### 索引

- `idx_emperor_sarcasm_logs_court_user` - 朝堂和用户组合索引
- `idx_emperor_sarcasm_logs_viewed` - 查看状态索引
- `idx_minister_log_progress_court_user` - 进度表组合索引

#### 路由层 (`backend/src/routes/sarcasm-logs.ts`)

**API 端点**：

1. **POST /api/logs/create** - 创建日志
   ```
   请求体：
   {
     court_id: string,
     user_id: string,
     log_type: string,
     log_message: string,
     related_data?: object
   }
   
   响应：
   {
     success: boolean,
     message: string,
     logId: string
   }
   ```

2. **GET /api/logs/unviewed/:courtId/:userId** - 获取未查看日志
   ```
   查询参数：limit (默认50)
   
   响应：
   {
     success: boolean,
     unviewedLogs: SarcasmLog[],
     unviewedCount: number
   }
   ```

3. **GET /api/logs/history/:courtId/:userId** - 获取日志历史
   ```
   查询参数：limit (默认50), offset (默认0)
   
   响应：
   {
     success: boolean,
     logs: SarcasmLog[],
     total: number
   }
   ```

4. **POST /api/logs/mark-viewed** - 标记日志为已查看
   ```
   请求体：
   {
     courtId: string,
     userId: string,
     logIds: string[]
   }
   
   响应：
   {
     success: boolean,
     message: string
   }
   ```

5. **GET /api/logs/progress/:courtId/:userId** - 获取日志进度
   ```
   响应：
   {
     success: boolean,
     progress: LogProgress
   }
   ```

6. **GET /api/logs/unviewed-count/:courtId/:userId** - 获取未查看日志数量
   ```
   响应：
   {
     success: boolean,
     unviewedCount: number
   }
   ```

#### Home 页面集成 (`frontend/src/pages/Home.tsx`)

**状态管理**：
```typescript
const [showLogModal, setShowLogModal] = useState(false);
const [unviewedLogs, setUnviewedLogs] = useState<any[]>([]);
```

**生命周期**：
```typescript
// 在组件挂载时检查未查看日志
useEffect(() => {
  const checkUnviewedLogs = async () => {
    if (!currentCourt?.id || !user?.id) return;
    
    try {
      const response = await logsAPI.getUnviewed(currentCourt.id, user.id);
      if (response.data.unviewedLogs && response.data.unviewedLogs.length > 0) {
        setUnviewedLogs(response.data.unviewedLogs);
        setShowLogModal(true);
      }
    } catch (error) {
      console.error('获取未查看日志失败:', error);
    }
  };
  
  checkUnviewedLogs();
}, [currentCourt?.id, user?.id]);
```

**确认处理**：
```typescript
const handleLogConfirm = async () => {
  if (!currentCourt?.id || !user?.id || unviewedLogs.length === 0) return;
  
  try {
    const logIds = unviewedLogs.map(log => log.id);
    await logsAPI.markAsViewed(currentCourt.id, user.id, logIds);
    setShowLogModal(false);
    setUnviewedLogs([]);
    showToast('日志已查看');
  } catch (error) {
    console.error('标记日志为已查看失败:', error);
    showToast('标记日志失败', 'error');
  }
};
```

## 使用流程

### 1. 创建日志

在游戏逻辑中调用相应的日志函数：

```typescript
// 任务分配时
await logTaskAssignment(courtId, ministerId, ministerName, taskName, taskId);

// 任务失败时
await logTaskFailure(courtId, ministerId, ministerName, taskName, taskId, score);

// 任务成功时
await logTaskSuccess(courtId, ministerId, ministerName, taskName, taskId, score);

// 排名变化时
await logRankingChange(courtId, ministerId, ministerName, oldRank, newRank, oldScore, newScore);

// 皇帝嘲笑时
await logEmperorMocking(courtId, ministerId, ministerName, mockingMessage);

// 惩罚时
await logPunishment(courtId, ministerId, ministerName, punishmentType);

// 奖励时
await logReward(courtId, ministerId, ministerName, rewardType, rewardAmount);
```

### 2. 大臣登录游戏

1. 前端检查是否有未查看日志
2. 如果有，显示 LogModal
3. 大臣必须查看所有日志才能继续游戏
4. 点击确认后，标记所有日志为已查看

### 3. 下次登录

1. 只显示上次确认之后的新日志
2. 重复流程

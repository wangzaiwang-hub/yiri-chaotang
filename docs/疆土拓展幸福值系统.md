# 疆土拓展幸福值系统

## 功能概述
实现了一个完整的幸福值系统，通过与大臣的交互积累幸福值，达到一定程度可以解锁新的场景疆域。

## 核心机制

### 1. 幸福值获取
每次与大臣交互都会增加幸福值：
- **发布任务**：+10-20 随机幸福值
- **准奏**：+10-20 随机幸福值
- **驳回**：+10-20 随机幸福值

### 2. 场景解锁系统
每个场景需要达到特定的幸福值才能解锁：
- **华夏**：0（初始场景）
- **东瀛**：100
- **高丽**：200
- **安南**：350
- **暹罗**：550
- **大食**：800

解锁所需幸福值递增，越后面的场景越难解锁。

### 3. 进度条显示

#### 左侧：百姓状态图片
根据当前进度显示不同的百姓状态：
- **0-20%**：qiong.png（穷困）
- **20-40%**：qiang.png（小康）
- **40-100%**：fu.png（富裕）

#### 中间：进度条
- 显示"疆土拓展"标题
- 进度条显示当前场景到下一场景的进度（0-100%）
- 显示"百姓幸福度：X%"
- 进度条有闪光动画效果

#### 右侧：下一个国家名字
- 当进度达到 100% 时，右侧显示下一个可解锁的国家名字
- 例如："→ 东瀛"
- 配合闪烁动画提示可以解锁

### 4. 解锁交互

#### 未达到 100%
点击进度条打开场景选择弹窗，显示所有已解锁的场景，可以切换背景。

#### 达到 100%
- 进度条边框闪烁（animate-pulse-glow）
- 右侧显示下一个国家名字并闪烁
- 点击进度条直接触发解锁动画
- 显示翻牌动画，解锁新场景
- 解锁后自动切换到新场景背景

### 5. 进度计算逻辑

```typescript
// 当前场景的进度（0-100）
const getCurrentProgress = () => {
  if (!nextScene) return 100; // 已经是最后一个场景
  const currentRequired = currentScene.requiredHappiness;
  const nextRequired = nextScene.requiredHappiness;
  const range = nextRequired - currentRequired;
  const progress = ((happiness - currentRequired) / range) * 100;
  return Math.min(Math.max(progress, 0), 100);
};
```

例如：
- 当前场景：华夏（0）
- 下一场景：东瀛（100）
- 当前幸福值：60
- 进度：(60 - 0) / (100 - 0) * 100 = 60%

### 6. 数据持久化
所有数据保存在 localStorage：
- `happiness`：当前幸福值
- `unlockedScenes`：已解锁的场景列表
- `currentSceneIndex`：当前场景索引
- `currentBackground`：当前背景场景 ID

## UI 设计

### 进度条样式
- 位置：页面顶部居中（top-6, left-1/2）
- 背景：琥珀色/橙色渐变，半透明
- 边框：金色，hover 时高亮
- 宽度：约 400px（包含图片和文字）
- 高度：自适应内容

### 动画效果
1. **闪光动画**：进度条内部有从左到右的闪光效果
2. **脉冲动画**：达到 100% 时整个进度条闪烁
3. **翻牌动画**：解锁新场景时的 3D 翻转效果

### 响应式设计
- 小人图片：40x40px
- 进度条：280px 宽
- 文字：使用 text-chinese-elegant 字体
- 颜色：金色/琥珀色主题

## 游戏平衡

### 幸福值获取速度
- 每次交互：10-20 幸福值
- 平均每次：15 幸福值

### 解锁所需交互次数
- 东瀛：100 / 15 ≈ 7 次交互
- 高丽：200 / 15 ≈ 13 次交互
- 安南：350 / 15 ≈ 23 次交互
- 暹罗：550 / 15 ≈ 37 次交互
- 大食：800 / 15 ≈ 53 次交互

### 设计理念
- 前期快速解锁，给玩家成就感
- 后期需要更多投入，增加游戏深度
- 每次解锁都有视觉反馈和仪式感

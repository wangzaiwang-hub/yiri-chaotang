# TTS 播放问题调试指南

## 问题描述
显示"播放中"指示器，但听不到声音。

## 可能的原因

### 1. 浏览器自动播放策略 ⚠️
现代浏览器（Chrome、Safari、Firefox）默认阻止自动播放音频，需要用户交互。

**解决方案**：
- 已添加手动播放按钮
- 当自动播放被阻止时，会显示蓝色"点击播放"按钮
- 点击按钮即可播放语音

### 2. 音量设置
检查系统和浏览器音量是否正常。

**检查步骤**：
1. 检查系统音量是否静音
2. 检查浏览器标签页是否静音（右键点击标签页）
3. 打开浏览器开发者工具 (F12)，查看 Console 日志

### 3. CORS 跨域问题
SecondMe 返回的音频 URL 可能有跨域限制。

**已处理**：
- 代码中已添加 `audio.crossOrigin = 'anonymous'`
- 如果仍有问题，查看 Console 是否有 CORS 错误

### 4. 音频 URL 失效
SecondMe 返回的音频 URL 可能有时效性。

**检查方法**：
1. 打开浏览器开发者工具 (F12)
2. 查看 Console 中的 `🔊 音频 URL:` 日志
3. 复制 URL 在新标签页打开，看是否能播放
4. 如果无法打开，说明 URL 已失效或有访问限制

## 调试步骤

### 第 1 步：打开浏览器开发者工具
按 `F12` 或右键点击页面 → 检查

### 第 2 步：查看 Console 日志
在游戏中触发语音播放，观察 Console 输出：

```
🔊 开始生成 TTS 语音...
🔊 清理后的文本: ...
🔊 TTS API 响应: ...
🔊 音频 URL: https://...
🔊 开始加载音频...
🔊 音频可以播放了
🔊 播放命令已发送
```

### 第 3 步：检查错误信息
如果有错误，会显示：

**自动播放被阻止**：
```
🔊 播放被阻止: NotAllowedError
🔊 浏览器阻止了自动播放，显示手动播放按钮
```
→ 点击蓝色"点击播放"按钮

**音频加载失败**：
```
🔊 音频播放失败: ...
🔊 音频错误详情: { code: 4, message: "MEDIA_ELEMENT_ERROR: ..." }
```
→ 检查网络连接，或音频 URL 是否有效

**TTS API 调用失败**：
```
🔊 TTS 生成失败: ...
```
→ 检查后端服务是否正常，用户是否设置了语音

### 第 4 步：手动测试音频 URL
1. 从 Console 复制音频 URL
2. 在新标签页打开
3. 看是否能播放

如果能播放 → 浏览器自动播放策略问题，点击"点击播放"按钮
如果不能播放 → 音频 URL 问题，检查 SecondMe 设置

## 常见问题

### Q1: 为什么第一次没声音，第二次就有了？
A: 浏览器的自动播放策略。第一次需要用户交互（点击播放按钮），之后浏览器会记住允许播放。

### Q2: 如何允许自动播放？
A: 
- Chrome: 地址栏左侧 → 网站设置 → 声音 → 允许
- Safari: Safari 菜单 → 偏好设置 → 网站 → 自动播放 → 允许所有自动播放
- Firefox: 地址栏左侧 → 权限 → 自动播放 → 允许音频和视频

### Q3: 音频 URL 能在浏览器打开，但游戏中不播放？
A: 可能是 CORS 问题。查看 Console 是否有跨域错误。已在代码中添加 `crossOrigin = 'anonymous'`。

### Q4: 显示"语音生成失败"？
A: 
1. 检查后端服务是否运行
2. 检查用户是否在 SecondMe 中设置了语音
3. 查看 Console 中的详细错误信息

## 技术细节

### 音频播放流程
```
1. 大臣回复任务
2. 打字机效果显示文本
3. 打字完成 → 调用 playTTS()
4. 清理文本（移除 Markdown 和工具调用标记）
5. 调用 TTS API 生成语音
6. 获取音频 URL
7. 创建 Audio 元素
8. 尝试自动播放
   ├─ 成功 → 显示"播放中"指示器
   └─ 失败 → 显示"点击播放"按钮
9. 播放完成 → 隐藏指示器
```

### 浏览器自动播放策略
- 需要用户交互（点击、触摸等）
- 或者网站已被用户添加到主屏幕
- 或者用户在该网站上有媒体播放历史

### Audio 元素配置
```javascript
audio.crossOrigin = 'anonymous'; // CORS
audio.preload = 'auto';          // 预加载
audio.volume = 1.0;              // 最大音量
```

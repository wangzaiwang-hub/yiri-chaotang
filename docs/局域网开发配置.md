# 局域网开发配置说明

## 当前配置

- 前端：http://192.168.124.47:3000
- 后端：http://192.168.124.47:3001
- 回调 URL：http://192.168.124.47:3001/api/auth/callback

## ⚠️ 重要：更新 SecondMe 回调 URL

由于 SecondMe OAuth 需要验证回调 URL，你需要在 SecondMe 开发者平台添加局域网回调地址：

### 步骤：

1. **访问 SecondMe 开发者平台**
   - 打开：https://develop.second-me.cn/integrations/list
   - 找到你的应用（Client ID: 9b8f2a1b-905d-48c3-8db3-aa96085f8148）

2. **添加回调 URL**
   - 在"重定向 URI"中添加：
     ```
     http://192.168.124.47:3001/api/auth/callback
     ```
   - 保留原有的 ngrok 地址（如果还需要）
   - 保存配置

3. **测试登录**
   - 访问：http://192.168.124.47:3000
   - 点击登录
   - 应该能正常跳转并回调

## 问题排查

### 如果登录后跳转到 localhost

这是因为浏览器缓存了旧的配置。解决方法：

1. 清除浏览器缓存
2. 使用无痕模式
3. 强制刷新（Cmd+Shift+R）

### 如果提示"回调 URL 不匹配"

检查 SecondMe 平台的回调 URL 配置是否包含：
```
http://192.168.124.47:3001/api/auth/callback
```

### 如果其他设备无法访问

1. 确认防火墙允许 3000 和 3001 端口
2. 确认所有设备在同一局域网
3. 尝试 ping 192.168.124.47

## 切换回 ngrok（如果需要）

如果需要切换回 ngrok 进行外网测试：

1. 启动 ngrok：
   ```bash
   ngrok http 3001
   ```

2. 更新 `backend/.env`：
   ```
   BASE_URL=https://你的ngrok域名.ngrok-free.dev
   FRONTEND_URL=https://你的ngrok域名.ngrok-free.dev
   ```

3. 更新 `frontend/.env`：
   ```
   VITE_API_URL=https://你的ngrok域名.ngrok-free.dev
   ```

4. 重启前后端服务

## 生产环境部署

部署到 Railway + Vercel 时：

1. Railway 后端环境变量：
   ```
   BASE_URL=https://你的项目.up.railway.app
   FRONTEND_URL=https://你的项目.vercel.app
   ```

2. Vercel 前端环境变量：
   ```
   VITE_API_URL=https://你的项目.up.railway.app
   ```

3. SecondMe 回调 URL：
   ```
   https://你的项目.up.railway.app/api/auth/callback
   ```

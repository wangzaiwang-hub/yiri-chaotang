# 文件库说明

## 文件存储位置

所有工具（内置工具和 MCP 工具）创建的文件都统一存储在：

```
backend/outputs/
```

## 文件库功能

在朝堂主页面，点击左上角的"📁 文件库"按钮，可以：

- 查看所有生成的文件
- 下载文件
- 删除文件

## 工具类型

### 1. 内置工具

这些工具直接在后端实现，创建的文件保存在 `backend/outputs/`：

- `createDocument`: 创建 Markdown 文档
- `createPresentation`: 创建 HTML 演示文稿
- `createTodoList`: 创建待办清单
- `analyzeData`: 数据分析（生成报告）
- `searchWeb`: 网络搜索（模拟）

### 2. MCP 工具

通过 MCP 协议连接的外部工具，现在也配置为在 `backend/outputs/` 目录工作：

- `read_file`: 读取文件
- `write_file`: 写入文件
- `create_directory`: 创建目录
- `list_directory`: 列出目录内容
- `move_file`: 移动文件
- `search_files`: 搜索文件
- `get_file_info`: 获取文件信息
- 天气查询工具等

## 配置修改

**重要**：修改了 MCP 文件系统服务器的工作目录配置：

```typescript
// 之前：工作在 backend/ 根目录
args: ['-y', '@modelcontextprotocol/server-filesystem', process.cwd()]

// 现在：工作在 backend/outputs/ 目录
args: ['-y', '@modelcontextprotocol/server-filesystem', `${process.cwd()}/outputs`]
```

这样 MCP 工具创建的文件也会出现在文件库中。

## 重启后端

修改配置后，需要重启后端服务才能生效：

```bash
# 停止当前运行的后端
# 然后重新启动
cd backend
npm run dev
```

## 测试

1. 在朝堂中发布任务："帮我创建一个测试文件 hello.txt，内容是 Hello World"
2. 等待大臣回复
3. 点击"📁 文件库"按钮
4. 应该能看到新创建的文件

## 文件清理

- 文件库中的文件可以随时删除
- 删除操作会立即生效
- 建议定期清理不需要的文件

## 注意事项

- MCP 文件系统工具只能在 `outputs/` 目录及其子目录中操作
- 这是出于安全考虑，防止误操作其他重要文件
- 如果需要访问其他目录，需要修改 MCP 配置

### 气泡显示逻辑

1. **消息气泡（紫色）**：
   - 显示在被点击的大臣（whisperTarget）上方
   - 打字机效果显示用户输入的消息
   - whisperPhase 为 'message_bubble' 时显示

2. **回复气泡（粉色）**：
   - 显示在被点击的大臣上方
   - 打字机效果显示 AI 生成的回复
   - whisperPhase 为 'reply_bubble' 时显示

3. **等待动画**：
   - whisperPhase 为 'waiting_reply' 时可以显示 thinking.gif

### 完整流程

```
1. 大臣A点击大臣B
   ↓
2. 弹出输入框，输入消息
   ↓
3. 关闭弹窗，在大臣B上方显示紫色气泡
   ↓
4. 打字机效果显示消息内容
   ↓
5. 消息打完，等待2秒
   ↓
6. 调用API，显示thinking动画（可选）
   ↓
7. API返回，在大臣B上方显示粉色气泡
   ↓
8. 打字机效果显示回复内容
   ↓
9. 回复打完，等待3秒
   ↓
10. 气泡淡出，流程结束
```

## 编译和测试

### 1. 编译后端
```bash
cd backend
npm run build
```

### 2. 重启后端
停止并重新启动后端服务

### 3. 运行数据库迁移
在 Supabase 中执行 `database/add_whispers.sql`

### 4. 测试流程
1. 以大臣身份登录
2. 点击另一个大臣的小人图片
3. 输入消息
4. 观察气泡动画
5. 查看AI生成的回复

## 注意事项

1. 气泡颜色区分：消息用紫色，回复用粉色
2. 气泡位置：显示在对应大臣上方
3. 打字机速度：50ms/字符
4. 等待时间：消息后2秒，回复后3秒
5. 皇帝可以看到所有悄悄话（通过 WebSocket 广播）

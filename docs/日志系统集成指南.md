# 皇帝毒舌日志系统 - 集成指南

## 快速开始

### 1. 导入日志函数

在需要记录日志的文件中导入：

```typescript
import {
  logTaskAssignment,
  logTaskFailure,
  logTaskSuccess,
  logRankingChange,
  logEmperorMocking,
  logPunishment,
  logReward
} from '../services/sarcasm-logs.service';
```

### 2. 在游戏逻辑中调用

#### 任务分配时

```typescript
// 在任务创建或分配时调用
await logTaskAssignment(
  courtId,           // 朝堂ID
  ministerId,        // 大臣ID
  ministerName,      // 大臣名字
  taskName,          // 任务名称
  taskId             // 任务ID
);
```

#### 任务失败时

```typescript
// 在任务被拒绝或失败时调用
await logTaskFailure(
  courtId,           // 朝堂ID
  ministerId,        // 大臣ID
  ministerName,      // 大臣名字
  taskName,          // 任务名称
  taskId,            // 任务ID
  score              // 失败分数
);
```

#### 任务成功时

```typescript
// 在任务完成时调用
await logTaskSuccess(
  courtId,           // 朝堂ID
  ministerId,        // 大臣ID
  ministerName,      // 大臣名字
  taskName,          // 任务名称
  taskId,            // 任务ID
  score              // 成功分数
);
```

#### 排名变化时

```typescript
// 在排名更新时调用
await logRankingChange(
  courtId,           // 朝堂ID
  ministerId,        // 大臣ID
  ministerName,      // 大臣名字
  oldRank,           // 旧排名
  newRank,           // 新排名
  oldScore,          // 旧无能指数
  newScore           // 新无能指数
);
```

#### 皇帝嘲笑时

```typescript
// 在皇帝嘲笑大臣时调用
await logEmperorMocking(
  courtId,           // 朝堂ID
  ministerId,        // 大臣ID
  ministerName,      // 大臣名字
  mockingMessage     // 嘲笑内容
);
```

#### 惩罚时

```typescript
// 在执行惩罚时调用
await logPunishment(
  courtId,           // 朝堂ID
  ministerId,        // 大臣ID
  ministerName,      // 大臣名字
  punishmentType     // 惩罚类型: 'virtual' | 'public' | 'salary'
);
```

#### 奖励时

```typescript
// 在发放奖励时调用
await logReward(
  courtId,           // 朝堂ID
  ministerId,        // 大臣ID
  ministerName,      // 大臣名字
  rewardType,        // 奖励类型: 'salary' | 'promotion' | 'reward'
  rewardAmount       // 奖励数额（可选）
);
```

## 集成位置

### 1. 任务路由 (`backend/src/routes/tasks.ts`)

#### 任务创建时
```typescript
// 在 POST /tasks 中
await logTaskAssignment(courtId, ministerId, ministerName, taskName, taskId);
```

#### 任务拒绝时
```typescript
// 在 POST /tasks/:id/reject 中
await logTaskFailure(courtId, ministerId, ministerName, taskName, taskId, 0);
```

#### 任务批准时
```typescript
// 在 POST /tasks/:id/approve 中
await logTaskSuccess(courtId, ministerId, ministerName, taskName, taskId, score);
```

### 2. 排名系统

#### 排名更新时
```typescript
// 在排名计算逻辑中
await logRankingChange(courtId, ministerId, ministerName, oldRank, newRank, oldScore, newScore);
```

### 3. 惩罚系统

#### 执行惩罚时
```typescript
// 在 POST /tasks/punish-minister 中
await logPunishment(courtId, ministerId, ministerName, punishmentType);
```

### 4. 奖励系统

#### 发放奖励时
```typescript
// 在奖励逻辑中
await logReward(courtId, ministerId, ministerName, rewardType, rewardAmount);
```

## 前端显示

### 自动显示日志

大臣登录时，前端会自动：

1. 检查是否有未查看日志
2. 如果有，显示 LogModal
3. 大臣必须查看所有日志才能继续游戏

### 手动检查日志

```typescript
// 获取未查看日志
const response = await logsAPI.getUnviewed(courtId, userId);
const unviewedLogs = response.data.unviewedLogs;

// 获取日志历史
const historyResponse = await logsAPI.getHistory(courtId, userId, 50, 0);
const allLogs = historyResponse.data.logs;

// 获取日志进度
const progressResponse = await logsAPI.getProgress(courtId, userId);
const progress = progressResponse.data.progress;
```

## 讽刺消息自定义

### 修改讽刺消息

编辑 `backend/src/services/sarcasm-logs.service.ts` 中的消息数组：

```typescript
// 例如，修改任务分配消息
const sarcasticMessages = [
  `皇帝让${ministerName}去${taskName}，${ministerName}连屁都不敢放，乖乖接受了`,
  `皇帝让${ministerName}去${taskName}，${ministerName}虽然很不情愿，但还是接受了`,
  // 添加更多消息...
];
```

### 添加新的日志类型

1. 在 `sarcasm-logs.service.ts` 中添加新函数
2. 在 `sarcasm-logs.ts` 路由中添加新端点（如需要）
3. 在游戏逻辑中调用新函数

## 常见问题

### Q: 日志什么时候显示？
A: 大臣登录游戏时，如果有未查看的日志，会自动显示 LogModal。

### Q: 大臣可以跳过日志吗？
A: 不可以。必须查看所有日志才能点击确认按钮。

### Q: 日志会永久保存吗？
A: 是的。所有日志都会保存在数据库中，可以随时查看历史。

### Q: 如何自定义讽刺消息？
A: 编辑 `backend/src/services/sarcasm-logs.service.ts` 中的消息数组。

### Q: 如何添加新的日志类型？
A: 在服务中添加新函数，然后在游戏逻辑中调用。

## 性能优化

### 索引

数据库已创建以下索引以优化查询性能：

- `idx_emperor_sarcasm_logs_court_user` - 朝堂和用户组合索引
- `idx_emperor_sarcasm_logs_viewed` - 查看状态索引
- `idx_minister_log_progress_court_user` - 进度表组合索引

### 分页

获取日志时使用分页：

```typescript
// 获取前50条未查看日志
const response = await logsAPI.getUnviewed(courtId, userId, 50);

// 获取历史日志，分页
const historyResponse = await logsAPI.getHistory(courtId, userId, 50, 0);
```

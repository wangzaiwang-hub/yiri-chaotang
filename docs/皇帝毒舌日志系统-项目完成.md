## 🎯 核心功能

### 1. 日志记录系统

**功能：** 记录皇帝对大臣的每一个操作

**支持的日志类型：**
- 任务分配
- 任务失败
- 任务成功
- 排名变化
- 皇帝嘲笑
- 惩罚
- 奖励

**毒舌语言：** 每个日志都用讽刺和毒舌的语言描述

### 2. 日志查询系统

**功能：** 获取大臣的日志

**支持的查询：**
- 获取未查看日志
- 获取日志历史
- 获取日志进度
- 获取未查看数量

### 3. 日志管理系统

**功能：** 管理大臣的日志查看状态

**支持的操作：**
- 标记日志为已查看
- 追踪查看进度
- 自动更新未查看计数

## 💡 系统设计亮点

### 1. 强制回顾机制

大臣进游戏时必须看完所有日志才能继续，无法逃避自己的无能。

### 2. 毒舌讽刺语言

每个日志都用讽刺和毒舌的语言描述，增强气人效果。

### 3. 持久记录

所有操作都被记录在数据库中，无法改写历史。

### 4. 怨气值驱动

看完日志后怨气值增加，推动游戏循环。

### 5. 进度追踪

自动追踪大臣的日志查看进度，下次进游戏从上次确认之后开始。

## 🎯 使用场景

### 场景 1：大臣进游戏

```
1. 检查未查看日志
2. 如果有日志，显示弹窗
3. 大臣查看所有日志
4. 点击确认
5. 标记为已查看
6. 怨气值增加
7. 继续游戏
```

### 场景 2：皇帝分配任务

```
1. 皇帝分配任务
2. 创建日志："皇帝让XXX去做什么，XXX连屁都不敢放"
3. 日志保存到数据库
4. 大臣下次进游戏时看到
```

### 场景 3：任务失败

```
1. 大臣任务失败
2. 创建日志："XXX失败了，皇帝看了一眼就摇头了"
3. 排名下降
4. 创建日志："XXX的排名从第5位下降到第4位，XXX的脸都绿了"
5. 大臣下次进游戏时看到
```

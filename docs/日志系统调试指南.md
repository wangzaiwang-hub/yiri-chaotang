# 皇帝毒舌日志系统 - 调试指南

## 问题：大臣刷新界面看不到日志

### 排查步骤

#### 1. 检查数据库中是否有日志数据

```bash
cd backend
node test-check-logs.js
```

**预期结果**:
- 应该看到日志数据
- 应该有未查看的日志

**如果没有日志数据**:
```bash
# 为指定朝堂创建测试日志
node test-create-logs-for-court.js
```

#### 2. 检查前端是否正确调用 API

打开浏览器开发者工具（F12），查看 Console 标签：

**应该看到的日志**:
```
🔍 检查未查看日志...
✅ 获取未查看日志成功
   未查看日志数: X
```

**如果看到错误**:
- 检查 API URL 配置
- 检查网络请求是否成功
- 检查返回的数据格式

#### 3. 检查角色判断

在浏览器控制台输入：
```javascript
// 检查当前用户角色
console.log('isEmperor:', isEmperor);
console.log('user:', user);
console.log('currentCourt:', currentCourt);
```

**预期结果**:
- 大臣端：`isEmperor` 应该是 `false`
- 皇帝端：`isEmperor` 应该是 `true`

#### 4. 检查 API 请求

在浏览器开发者工具的 Network 标签中：

1. 刷新页面
2. 查找 `/api/logs/unviewed/` 请求
3. 检查请求状态码（应该是 200）
4. 检查响应数据

**预期响应**:
```json
{
  "success": true,
  "unviewedLogs": [
    {
      "id": "...",
      "log_message": "...",
      "log_type": "...",
      ...
    }
  ],
  "unviewedCount": 3
}
```

#### 5. 检查前端状态

在浏览器控制台输入：
```javascript
// 检查日志状态
console.log('showLogModal:', showLogModal);
console.log('unviewedLogs:', unviewedLogs);
```

**预期结果**:
- `showLogModal` 应该是 `true`（如果有未查看日志）
- `unviewedLogs` 应该是一个数组，包含日志数据

### 常见问题

#### 问题 1: 数据库中没有日志

**原因**: 游戏逻辑中还没有添加日志创建的调用

**解决方案**:
1. 使用测试脚本创建测试日志：
   ```bash
   cd backend
   node test-create-logs-for-court.js
   ```

2. 或者在游戏逻辑中添加日志创建调用（参考集成指南）

#### 问题 2: API 请求失败

**原因**: 
- 后端服务未启动
- API URL 配置错误
- 路由未注册

**解决方案**:
1. 检查后端服务是否运行：
   ```bash
   cd backend
   npm start
   ```

2. 检查 `frontend/.env` 中的 `VITE_API_URL` 配置

3. 检查 `backend/src/index.ts` 中是否注册了日志路由：
   ```typescript
   app.use('/api/logs', sarcasmLogsRoutes);
   ```

#### 问题 3: 角色判断错误

**原因**: `isEmperor` 判断逻辑错误

**解决方案**:
检查 `frontend/src/pages/Home.tsx` 中的代码：
```typescript
const isEmperor = emperor?.user_id === user?.id;

// 检查未查看的日志（只在大臣端触发）
useEffect(() => {
  const checkUnviewedLogs = async () => {
    // 只有大臣才检查日志
    if (!currentCourt?.id || !user?.id || isEmperor) return;
    // ...
  };
  // ...
}, [currentCourt?.id, user?.id, isEmperor]);
```

#### 问题 4: 前端状态未更新

**原因**: 
- useEffect 依赖项不正确
- 状态更新时机错误

**解决方案**:
1. 检查 useEffect 依赖项是否包含 `isEmperor`
2. 检查 `ranking` 数据是否正确加载
3. 尝试刷新页面

#### 问题 5: LogModal 不显示

**原因**: 
- `showLogModal` 状态未设置为 `true`
- `unviewedLogs` 数组为空
- LogModal 组件渲染条件不满足

**解决方案**:
1. 检查 `showLogModal` 和 `unviewedLogs` 状态
2. 检查 LogModal 组件的 `isOpen` prop
3. 检查 CSS 样式是否正确

### 调试技巧

#### 1. 添加调试日志

在 `frontend/src/pages/Home.tsx` 中添加：
```typescript
useEffect(() => {
  const checkUnviewedLogs = async () => {
    console.log('🔍 检查未查看日志...', {
      courtId: currentCourt?.id,
      userId: user?.id,
      isEmperor
    });

    if (!currentCourt?.id || !user?.id || isEmperor) {
      console.log('⏭️ 跳过日志检查:', {
        hasCourt: !!currentCourt?.id,
        hasUser: !!user?.id,
        isEmperor
      });
      return;
    }
    
    try {
      console.log('📡 调用 API 获取未查看日志...');
      const response = await logsAPI.getUnviewed(currentCourt.id, user.id);
      console.log('✅ API 响应:', response.data);
      
      if (response.data.unviewedLogs && response.data.unviewedLogs.length > 0) {
        console.log('📬 找到未查看日志:', response.data.unviewedLogs.length);
        setUnviewedLogs(response.data.unviewedLogs);
        setShowLogModal(true);
      } else {
        console.log('📭 没有未查看日志');
      }
    } catch (error) {
      console.error('❌ 获取未查看日志失败:', error);
    }
  };
  
  checkUnviewedLogs();
}, [currentCourt?.id, user?.id, isEmperor]);
```

### 测试流程

#### 完整测试流程

1. **创建测试数据**
   ```bash
   cd backend
   node test-create-logs-for-court.js
   ```

2. **启动后端服务**
   ```bash
   cd backend
   npm start
   ```

3. **启动前端服务**
   ```bash
   cd frontend
   npm run dev
   ```

4. **以大臣身份登录**
   - 使用大臣账号登录
   - 进入有日志的朝堂

5. **检查日志显示**
   - 应该自动显示 LogModal
   - 可以查看所有日志
   - 可以点击确认

6. **验证日志标记**
   - 点击确认后，日志应该被标记为已查看
   - 再次刷新页面，不应该再显示这些日志

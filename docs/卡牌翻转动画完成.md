# 卡牌翻转动画实现完成

## 功能说明

实现了召唤分身的卡牌翻转抽卡效果，用户体验流畅自然。卡牌正面显示用户的头像、昵称和个人简介。

## 实现细节

### 1. 状态管理
- `showSummonModal`: 控制召唤弹窗显示（显示卡牌背面网格）
- `showCardFlip`: 控制全屏翻牌动画显示
- `selectedBot`: 存储被选中的分身信息（包含 id, nickname, avatar, bio）
- `isFlipping`: 控制翻转动画触发
- `isSummoning`: 召唤中状态

### 2. 动画流程
1. 用户点击"召唤分身"按钮
2. 显示弹窗，展示 5 列卡牌背面网格
3. 点击任意卡牌 → `handleCardClick(bot)`
4. 关闭弹窗，显示全屏黑色遮罩
5. 100ms 后触发翻转动画（`setIsFlipping(true)`）
6. 卡牌 3D 旋转 180 度（1 秒动画）
7. 显示卡牌正面和用户信息（头像、昵称、个人简介、按钮）
8. 用户可以确认召唤或取消

### 3. 3D 翻转效果
使用 CSS 3D transforms 实现：
- `perspective: 1000px` - 设置透视距离
- `transformStyle: preserve-3d` - 保持 3D 空间
- `transform: rotateY(180deg)` - Y 轴旋转 180 度
- `backfaceVisibility: hidden` - 隐藏背面
- `transition-transform duration-1000` - 1 秒过渡动画

### 4. 卡牌结构
```
全屏遮罩 (bg-black/90)
  └─ 透视容器 (perspective: 1000px)
      └─ 翻转容器 (rotateY)
          ├─ 卡牌背面 (backfaceVisibility: hidden)
          │   └─ card_back.png
          └─ 卡牌正面 (rotateY(180deg) + backfaceVisibility: hidden)
              ├─ card_front.png (背景)
              └─ 用户信息叠加层
                  ├─ 头像 (32x32 圆形)
                  ├─ 昵称 (3xl 字体)
                  ├─ 个人简介 (bio，小字体)
                  ├─ 描述 (SecondMe 虚拟分身)
                  └─ 按钮（确认/取消）
```

### 5. 用户信息渲染
- 用户信息提前渲染在卡牌正面
- 使用绝对定位叠加在卡牌背景图上
- 头像：32x32 圆形，紫色边框
- 昵称：3xl 字体，紫色
- 个人简介：小字体，紫色，居中显示，最大宽度限制
- 按钮：取消（灰色）和确认（紫粉渐变）

### 6. 个人简介集成
- 后端：`/api/courts/:id/available-bots` 接口返回 `id, nickname, avatar, bio`
- 前端：从 SecondMe API 获取的 bio 字段显示在卡牌上
- 如果用户没有设置 bio，则不显示该区域
- 简介文字使用中文古风字体，居中对齐，行高适中

### 7. 交互逻辑
- 确认召唤 → `handleConfirmSummon()` → 调用 API → 刷新页面
- 取消召唤 → `handleCancelSummon()` → 返回卡牌选择弹窗
- 召唤中禁用按钮，显示"召唤中..."

## 技术要点

1. **CSS 3D 变换**：使用 `rotateY` 实现翻转效果
2. **双面卡牌**：正反面都提前渲染，通过 `backfaceVisibility` 控制显示
3. **延迟触发**：100ms 延迟确保 DOM 渲染完成后再开始动画
4. **平滑过渡**：1 秒 transition 让翻转更自然
5. **信息叠加**：用户信息绝对定位在卡牌正面背景图上
6. **条件渲染**：只有当 bio 存在时才显示个人简介区域

## API 集成

### SecondMe 用户信息 API
- 端点：`GET /api/secondme/user/info`
- 返回字段：
  - `userId`: 用户 ID
  - `name`: 用户姓名
  - `email`: 用户邮箱
  - `avatar`: 头像 URL
  - `bio`: 个人简介 ⭐
  - `selfIntroduction`: 自我介绍
  - `profileCompleteness`: 资料完整度
  - `route`: 用户主页路由

### 后端实现
- 在用户登录时，从 SecondMe API 获取用户信息并保存 bio 到数据库
- `available-bots` 接口查询时包含 bio 字段
- 数据库 users 表已有 bio 字段

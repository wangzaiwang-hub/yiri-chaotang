# 召唤分身功能说明

## 功能概述

"召唤分身"功能允许皇帝在没有真实玩家在线时，召唤其他用户的 SecondMe 虚拟分身加入朝堂，实现单人游戏体验。

## 功能特点

1. **自动人机选择**：从数据库中随机选择不在当前朝堂的用户作为人机
2. **随机部门分配**：人机加入时自动随机分配到三省六部之一
3. **SecondMe 驱动**：人机使用真实用户的 SecondMe 虚拟分身，具有独特性格
4. **无需等待**：不需要等待好友在线，随时可以开始游戏

## 使用方法

### 前端操作

1. 在朝堂主界面，点击左上角的"召唤分身"按钮（紫色边框）
2. 弹出分身列表，显示所有可召唤的用户（最多 20 个）
3. 点击任意分身卡片，即可召唤该分身入朝
4. 召唤成功后，页面自动刷新，分身出现在朝堂中

### 按钮位置

左上角按钮从上到下：
- 招贤纳士（邀请真实玩家）
- **召唤分身**（邀请人机）← 新增
- 档案阁
- 退位让贤/告老还乡

### 前端实现

#### API 调用（`frontend/src/services/api.ts`）

```typescript
export const courtAPI = {
  // ... 其他方法
  getAvailableBots: (courtId: string, currentUserId: string) =>
    api.get(`/courts/${courtId}/available-bots?current_user_id=${currentUserId}`),
  summonBot: (courtId: string, userId: string) =>
    api.post(`/courts/${courtId}/summon-bot`, { user_id: userId }),
};
```

#### 状态管理（`frontend/src/pages/Home.tsx`）

```typescript
// 召唤分身相关状态
const [showSummonModal, setShowSummonModal] = useState(false);
const [availableBots, setAvailableBots] = useState<any[]>([]);
const [isSummoning, setIsSummoning] = useState(false);
```

#### 核心函数

```typescript
// 打开召唤分身弹窗
const handleOpenSummonModal = async () => {
  const response = await courtAPI.getAvailableBots(currentCourt.id, user.id);
  setAvailableBots(response.data.data || []);
  setShowSummonModal(true);
};

// 召唤分身
const handleSummonBot = async (botUserId: string, botNickname: string) => {
  await courtAPI.summonBot(currentCourt.id, botUserId);
  showToast(`已召唤 ${botNickname} 分身入朝`);
  window.location.reload();
};
```

## 随机部门分配

召唤分身时，系统会从以下 9 个部门中随机选择一个：

1. 中书省（zhongshu）
2. 门下省（menxia）
3. 尚书省（shangshu）
4. 户部（hubu）
5. 礼部（libu）
6. 兵部（bingbu）
7. 刑部（xingbu）
8. 工部（gongbu）
9. 吏部（libu_hr）

## 注意事项

1. **去重处理**：已在朝堂中的用户不会出现在可召唤列表中
2. **数量限制**：最多显示 20 个可召唤分身
3. **错误处理**：如果分身已在朝堂中，会提示"该分身已在朝堂中"
4. **自动刷新**：召唤成功后自动刷新页面，显示新成员
5. **权限控制**：只有皇帝可以看到"召唤分身"按钮

## 与"招贤纳士"的区别

| 功能 | 招贤纳士 | 召唤分身 |
|------|---------|---------|
| 对象 | 真实玩家 | SecondMe 虚拟分身 |
| 方式 | 生成邀请链接 | 直接选择召唤 |
| 部门 | 玩家自选 | 系统随机分配 |
| 在线要求 | 需要玩家在线接受邀请 | 无需在线，立即加入 |
| 按钮颜色 | 金色边框 | 紫色边框 |

## 未来优化方向

1. **智能推荐**：根据朝堂缺少的部门推荐合适的分身
2. **分身标记**：在界面上标识哪些是人机，哪些是真实玩家
3. **批量召唤**：一次性召唤多个分身
4. **分身管理**：可以解散（移除）人机成员
5. **性格匹配**：根据任务类型推荐合适性格的分身

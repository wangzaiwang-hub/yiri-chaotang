# 登录状态自动退出问题排查

## 问题描述

用户反馈经常自动退出登录状态。

## 可能原因

### 1. API 401 错误导致自动退出

**原因**：API 拦截器在收到 401 错误时会自动调用 `logout()` 并跳转到登录页。

**解决方案**：
- 已修改 API 拦截器，只在明确的认证错误时才退出登录
- 添加了详细的错误日志，方便排查问题

```typescript
// 修改后的逻辑
if (error.response?.status === 401) {
  const errorMessage = error.response?.data?.error || '';
  // 只有在明确的 token 错误时才退出
  if (errorMessage.includes('token') || errorMessage.includes('auth')) {
    logout();
  }
}
```

### 2. localStorage 被清除

**原因**：
- 浏览器隐私模式
- 浏览器清除缓存
- 第三方扩展清除数据

**检查方法**：
1. 打开浏览器开发者工具
2. 进入 Application > Local Storage
3. 查看是否有 `auth-storage` 键

### 3. Token 过期

**原因**：SecondMe 的 access_token 有过期时间

**解决方案**：
- 实现 token 刷新机制
- 使用 refresh_token 自动刷新

### 4. 网络请求失败

**原因**：
- 网络不稳定
- 后端服务重启
- API 请求超时

**当前问题**：登录页面的 `authAPI.getMe()` 失败会清除 token

## 调试方法

### 1. 查看浏览器控制台

打开浏览器开发者工具（F12），查看：
- Console 标签：查看错误日志
- Network 标签：查看 API 请求状态
- Application > Local Storage：查看存储的认证信息

### 2. 查看后端日志

后端日志会显示所有 API 请求和错误：
```bash
# 查看后端进程输出
# 在 Kiro 中使用 getProcessOutput 工具
```

### 3. 添加调试日志

在关键位置添加 `console.log`：
- API 拦截器
- 登录页面
- 认证状态变化

## 临时解决方案

如果频繁退出登录，可以：

1. **禁用自动退出**（不推荐）：
   - 注释掉 API 拦截器中的 `logout()` 调用
   - 只保留错误日志

2. **增加重试机制**：
   - API 请求失败时自动重试
   - 避免因临时网络问题导致退出

3. **使用 Session Storage**：
   - 将认证信息存储在 sessionStorage
   - 关闭浏览器标签页才会清除

## 长期解决方案

### 1. 实现 Token 刷新

```typescript
// 在 API 拦截器中
if (error.response?.status === 401) {
  // 尝试刷新 token
  const refreshToken = useAuthStore.getState().refreshToken;
  if (refreshToken) {
    try {
      const newToken = await refreshAccessToken(refreshToken);
      // 重试原请求
      return api.request(error.config);
    } catch {
      // 刷新失败，退出登录
      logout();
    }
  }
}
```

### 2. 心跳检测

定期检查认证状态：
```typescript
setInterval(async () => {
  try {
    await authAPI.getMe();
  } catch (error) {
    // 认证失败，提示用户重新登录
  }
}, 5 * 60 * 1000); // 每 5 分钟检查一次
```

### 3. 优雅降级

当认证失败时：
- 不立即退出登录
- 显示提示信息
- 允许用户手动刷新或重新登录

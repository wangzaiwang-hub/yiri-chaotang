# 部署步骤清单

## 准备工作

- [ ] 注册 Railway 账号：https://railway.app
- [ ] 注册 Vercel 账号：https://vercel.com
- [ ] 准备 Supabase 数据库（已有）
- [ ] 准备 SecondMe API 密钥（已有）
- [ ] 将代码推送到 GitHub

## 第一步：部署后端到 Railway

### 1.1 创建项目

- [ ] 访问 https://railway.app
- [ ] 点击 "New Project"
- [ ] 选择 "Deploy from GitHub repo"
- [ ] 选择你的仓库
- [ ] 选择 `backend` 目录作为根目录

### 1.2 配置环境变量

在 Railway 项目设置 → Variables 中添加：

- [ ] `NODE_ENV=production`
- [ ] `PORT=3001`
- [ ] `SUPABASE_URL=你的Supabase URL`
- [ ] `SUPABASE_ANON_KEY=你的Supabase匿名密钥`
- [ ] `SUPABASE_SERVICE_ROLE_KEY=你的Supabase服务密钥`
- [ ] `SECONDME_CLIENT_ID=你的SecondMe客户端ID`
- [ ] `SECONDME_CLIENT_SECRET=你的SecondMe客户端密钥`
- [ ] `SECONDME_API_KEY=你的SecondMe API密钥`
- [ ] `BASE_URL=https://你的项目名.up.railway.app`（先填临时值，部署后更新）
- [ ] `FRONTEND_URL=https://你的前端域名.vercel.app`（先填临时值，部署后更新）

### 1.3 部署

- [ ] 点击 "Deploy"
- [ ] 等待构建完成
- [ ] 记录 Railway 提供的域名（类似：`https://xxx.up.railway.app`）
- [ ] 更新 `BASE_URL` 环境变量为实际域名
- [ ] 测试后端健康检查：访问 `https://你的域名.up.railway.app/health`

## 第二步：部署前端到 Vercel

### 2.1 创建项目

- [ ] 访问 https://vercel.com/new
- [ ] 导入你的 GitHub 仓库
- [ ] 配置：
  - Root Directory: `frontend`
  - Framework Preset: `Vite`
  - Build Command: `npm run build`
  - Output Directory: `dist`

### 2.2 配置环境变量

在 Vercel 项目设置 → Environment Variables 中添加：

- [ ] `VITE_API_URL=https://你的Railway后端域名.up.railway.app`

### 2.3 部署

- [ ] 点击 "Deploy"
- [ ] 等待构建完成
- [ ] 记录 Vercel 提供的域名（类似：`https://xxx.vercel.app`）
- [ ] 测试前端页面：访问 `https://你的域名.vercel.app`

## 第三步：更新配置

### 3.1 更新 Railway 后端配置

- [ ] 在 Railway 项目设置中，将 `FRONTEND_URL` 更新为 Vercel 提供的前端域名
- [ ] 重新部署后端（或等待自动部署）

### 3.2 更新 SecondMe 配置

- [ ] 访问 SecondMe 开发者平台
- [ ] 更新回调 URL 为：`https://你的Railway域名.up.railway.app/api/auth/callback`
- [ ] 保存配置

### 3.3 更新 Supabase 配置

- [ ] 访问 Supabase 项目设置
- [ ] 进入 Authentication → URL Configuration
- [ ] 将前端域名添加到 Site URL
- [ ] 将前端域名添加到 Redirect URLs
- [ ] 保存配置

## 常见问题排查

### 后端无法访问

1. 检查 Railway 部署日志，查看是否有错误
2. 检查环境变量是否配置正确
3. 检查 Supabase 数据库连接是否正常

### 前端无法访问后端

1. 检查 `VITE_API_URL` 环境变量是否正确
2. 检查后端 CORS 配置是否包含前端域名
3. 检查浏览器控制台的网络请求

### 登录失败

1. 检查 SecondMe 回调 URL 是否正确
2. 检查 `BASE_URL` 和 `FRONTEND_URL` 是否正确
3. 检查 SecondMe API 密钥是否有效

### WebSocket 连接失败

1. 确认后端部署在 Railway（不是 Vercel）
2. 检查浏览器控制台的 WebSocket 连接日志
3. 检查后端日志，查看 WebSocket 服务是否启动

# 惩罚功能更新对比

## 更新内容

### 1. API 参数名统一

#### 之前 ❌
```typescript
// frontend/src/services/api.ts
punishMinister: (courtId, ministerId, emperorId, punishmentContent) =>
  api.post('/tasks/punish-minister', { 
    punishment_content: punishmentContent  // 错误的参数名
  })
```

#### 现在 ✅
```typescript
// frontend/src/services/api.ts
punishMinister: (courtId, ministerId, emperorId, punishmentTask) =>
  api.post('/tasks/punish-minister', { 
    punishment_task: punishmentTask  // 正确的参数名，与后端一致
  })
```

### 2. 惩罚弹窗文案更新

#### 之前 ❌
```
标题：惩罚大臣
说明：朕对此臣不满！让其在广场发布羞耻言论以示惩戒。
输入框占位符：例如：朕今日办事不力，愧对圣恩，特此自省...
提示：💡 此内容将以 XXX 的虚拟分身名义发布到 SecondMe Plaza 广场
```

**问题**：让皇帝直接写羞耻文案，不符合需求

#### 现在 ✅
```
标题：惩罚大臣
说明：朕对此臣不满！请描述惩罚原因，朕的虚拟分身会让其在广场自我检讨。
输入框占位符：例如：办事不力，屡次拖延，辜负圣恩...
提示：💡 AI 将根据你的描述自动生成羞耻文案，以 XXX 的虚拟分身名义发布到 SecondMe Plaza 广场
```

**改进**：
- 明确说明皇帝只需描述惩罚原因
- 强调 AI 会自动生成文案
- 占位符改为简短的原因描述
- 提示文字更清晰地说明流程

## 工作流程

### 用户操作流程
```
1. 皇帝点击大臣头像
   ↓
2. 弹出惩罚弹窗
   ↓
3. 皇帝输入惩罚原因（如：办事不力）
   ↓
4. 点击"执行惩罚"
   ↓
5. 前端调用 API，传递 punishment_task
   ↓
6. 后端接收 punishment_task
   ↓
7. AI 根据原因生成古风羞耻文案
   ↓
8. 以大臣名义发布到 Plaza
   ↓
9. 增加大臣怨气值 30 点
   ↓
10. 返回成功提示
```

### AI 生成示例

**输入（皇帝）**：
```
办事不力，屡次拖延，辜负圣恩
```

**输出（AI 生成的羞耻文案）**：
```
臣罪该万死！近日办事不力，屡次拖延，辜负圣上隆恩，实乃不忠不孝之徒。
臣愧对天地，愧对社稷，愧对圣上栽培之恩。今日当众自省，以儆效尤。
臣发誓痛改前非，勤勉尽责，以报圣恩于万一。恳请圣上及天下人见谅！
```

## 技术细节

### 后端 AI 生成逻辑
```typescript
const systemPrompt = `你是一位古代朝堂的大臣，刚刚被皇上惩罚。
你需要在广场发布一条羞耻的自我检讨言论。

要求：
1. 使用古风文言文风格
2. 表达深刻的自责和愧疚
3. 承认自己的过错
4. 请求皇上和天下人的原谅
5. 字数控制在 100-200 字
6. 语气要诚恳、卑微、羞愧
7. 不要使用 Markdown 格式

惩罚原因：${punishment_task}

请直接输出自我检讨的内容，不要有任何前缀或解释。`;
```

## 注意事项

1. **Plaza 准入**：大臣必须已激活 Plaza 准入才能发布帖子
2. **怨气值**：每次惩罚增加 30 点怨气值
3. **AI 生成**：文案由 AI 自动生成，无需人工编写
4. **古风风格**：生成的文案会使用古风文言文风格
5. **字数限制**：生成的文案控制在 100-200 字

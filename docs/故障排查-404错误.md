# 故障排查：召唤分身 404 错误

## 当前状态

✅ **后端 API 正常**：
```bash
curl "http://localhost:3001/api/courts/xxx/available-bots?current_user_id=test"
# 返回: {"code":0,"data":[]}
```

✅ **前端配置正确**：
```typescript
const API_BASE_URL = 'http://localhost:3001';
```

❌ **浏览器仍然报 404**：可能是缓存问题

## 解决方案

### 方案 1：清除浏览器缓存（推荐）

#### Chrome/Edge
1. 打开开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

或者：
1. 按 `Cmd + Shift + Delete`（Mac）或 `Ctrl + Shift + Delete`（Windows）
2. 选择"缓存的图片和文件"
3. 时间范围选择"全部时间"
4. 点击"清除数据"
5. 刷新页面

#### Safari
1. 按 `Cmd + Option + E` 清空缓存
2. 或者：Safari 菜单 → 偏好设置 → 高级 → 勾选"在菜单栏中显示开发菜单"
3. 开发菜单 → 清空缓存
4. 刷新页面

### 方案 2：使用无痕/隐私模式

1. 打开无痕窗口（Cmd+Shift+N 或 Ctrl+Shift+N）
2. 访问 `http://localhost:3000`
3. 测试召唤分身功能

### 方案 3：检查网络请求

1. 打开开发者工具（F12）
2. 切换到 Network 标签
3. 点击"召唤分身"按钮
4. 查看 `available-bots` 请求：
   - **URL 应该是**：`http://localhost:3001/api/courts/xxx/available-bots`
   - **如果是**：`https://backend-production-a216.up.railway.app/...` → 说明浏览器使用了缓存的旧代码

### 方案 4：重启前端开发服务器

```bash
# 停止前端服务器（Ctrl+C）
# 然后重新启动
cd frontend
npm run dev
```

### 方案 5：手动验证 API

在浏览器中直接访问：
```
http://localhost:3001/api/courts/d7473899-9582-4fbe-927b-cb396cb15815/available-bots?current_user_id=test
```

应该看到 JSON 响应，而不是 404。

## 常见问题

### Q: Network 标签显示请求到了 Railway 而不是 localhost？

A: 浏览器缓存了旧的 JavaScript 文件。解决方法：
1. 清空缓存
2. 硬刷新（Cmd+Shift+R）
3. 或使用无痕模式

### Q: 显示 CORS 错误？

A: 后端已经配置了 CORS 允许所有来源。如果还有 CORS 错误：
1. 检查后端是否真的在运行
2. 检查后端日志是否有错误
3. 重启后端服务器

### Q: 显示 "net::ERR_CONNECTION_REFUSED"？

A: 后端服务器没有运行。解决方法：
```bash
cd backend
npm run dev
```

### Q: API 返回空数组 `{"code":0,"data":[]}`？

A: 这是正常的！说明：
- API 工作正常
- 只是数据库中没有其他用户
- 或者所有用户都已经在朝堂中

要测试完整功能，需要：
1. 创建多个用户账号
2. 确保有用户不在当前朝堂中

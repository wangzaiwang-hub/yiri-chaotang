# 个人简介功能完成

## 功能说明

在卡牌翻转动画中添加了用户个人简介显示功能，从 SecondMe API 获取用户的 bio 信息并智能提取关键内容展示。

## 实现细节

### 1. 后端修改
- **文件**: `backend/src/routes/courts.ts`
- **接口**: `GET /api/courts/:id/available-bots`
- **修改**: 查询字段从 `id, nickname, avatar` 改为 `id, nickname, avatar, bio`
- **编译**: 运行 `npm run build` 重新编译 TypeScript
- **重启**: 重启后端服务使更改生效

### 2. 前端修改
- **文件**: `frontend/src/pages/Home.tsx`
- **位置**: 卡牌正面用户信息叠加层
- **显示逻辑**:
  1. 如果用户有 bio，显示简介区域
  2. 智能提取 bio 内容：
     - 优先提取"总体概述"部分
     - 如果没有，提取第一段内容
     - 移除 Markdown 标题标记（### xxx ###）
     - 限制显示 150 个字符，超出显示省略号
  3. 如果用户没有 bio，不显示该区域

### 3. 样式设计
- 字体大小: `text-xs` (小字体)
- 颜色: `text-purple-700` (紫色)
- 对齐: 居中对齐
- 最大宽度: `max-w-sm`
- 最大高度: `max-h-32` (可滚动)
- 内边距: `px-4`
- 行高: `leading-relaxed`

### 4. Bio 内容格式
SecondMe API 返回的 bio 格式：
```
### 背景与身份 ###
用户的背景信息...

### 性格特征 ###
用户的性格描述...

### 价值观 ###
用户的价值观...

### MBTI ###
INTJ/ENTP 等

### 总体概述 ###
综合描述用户的特点...
```

### 5. 智能提取算法
```javascript
// 1. 尝试提取"总体概述"部分
const overviewMatch = bio.match(/### 总体概述 ###\s*\n([\s\S]*?)(?:\n###|$)/);
if (overviewMatch && overviewMatch[1]) {
  return overviewMatch[1].trim().substring(0, 150) + '...';
}

// 2. 如果没有总体概述，提取第一段
const firstParagraph = bio.split('\n\n')[0].replace(/###.*###/g, '').trim();
return firstParagraph.substring(0, 150) + '...';
```

## 注意事项

1. **Bio 长度**: 限制显示 150 字符，避免卡牌内容过多
2. **滚动支持**: 如果内容超过 `max-h-32`，会显示滚动条
3. **条件渲染**: 只有当 bio 存在且不为 null 时才显示
4. **格式清理**: 自动移除 Markdown 标题标记
5. **优先级**: 优先显示"总体概述"，更准确地概括用户特点

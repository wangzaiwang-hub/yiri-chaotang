# 准奏补充说明功能

## 功能概述

在准奏时，皇帝可以添加补充说明，让大臣更好地理解执行要求。

## 使用流程

### 1. 点击"准"按钮
当大臣提交方案后，皇帝可以看到三个审批按钮：
- 结：结束对话
- 准：批准方案
- 驳：驳回方案

点击"准"按钮后，会弹出一个输入框。

### 2. 输入补充说明（可选）
在弹出的输入框中，皇帝可以：
- **留空直接准奏**：表示完全按大臣的方案执行
- **添加补充说明**：给出额外的指示或要求

示例补充说明：
```
按你的方案执行，注意控制成本，每周汇报一次进度
```

```
方案不错，但要加快进度，本月内完成
```

```
可以执行，记得和其他部门协调好
```

### 3. 确认准奏
点击"确定准奏"按钮后：
- 任务状态更新为"已批准"
- 大臣虚拟人收到准奏通知
- 补充说明会传递给大臣
- 大臣开始执行任务

#### 4. 修改准奏处理函数
```typescript
const handleApproveTask = async () => {
  setShowApprovalButtons(false);
  setShowApproveModal(false);
  
  try {
    await taskAPI.approveTask(selectedTaskForApproval.id, approvalFeedback);
    showToast('已准奏');
    setApprovalFeedback('');
    setSelectedTaskForApproval(null);
    refetchTasks();
  } catch (error) {
    console.error('准奏失败:', error);
    showToast('操作失败，请重试', 'error');
  }
};
```

## 使用场景

### 场景 1：完全认可方案
**操作**：留空直接点击"确定准奏"
**效果**：大臣收到"按你的方案，继续下一步"的指示

### 场景 2：方案可行但需要补充要求
**操作**：输入"注意控制成本，每周汇报进度"
**效果**：大臣收到补充要求，会在执行时考虑这些因素

### 场景 3：方案可行但需要调整细节
**操作**：输入"方案不错，但要加快进度，本月内完成"
**效果**：大臣会调整执行计划，加快进度

### 场景 4：方案可行但需要协调
**操作**：输入"可以执行，记得和户部协调好预算问题"
**效果**：大臣会注意部门协调

## 与驳回的区别

| 功能 | 准奏（带补充说明） | 驳回 |
|------|------------------|------|
| 任务状态 | 批准，继续执行 | 驳回，重新设计 |
| 大臣反应 | 按方案执行，考虑补充要求 | 重新设计方案 |
| 怨气值 | 不增加 | 增加 |
| 对话历史 | 保留，继续 | 保留，重新开始 |
| 使用场景 | 方案基本可行，需要微调 | 方案不可行，需要重做 |

## 测试方法

1. 启动前端和后端
2. 登录系统，进入朝堂
3. 给大臣下达任务
4. 等待大臣回复方案
5. 点击"准"按钮
6. 在弹出的输入框中输入补充说明（或留空）
7. 点击"确定准奏"
8. 观察大臣是否收到补充说明并调整执行方式

## 注意事项

1. **补充说明是可选的**：可以留空直接准奏
2. **补充说明会影响 AI 行为**：AI 会根据补充说明调整回复
3. **不要与驳回混淆**：准奏是批准，驳回是拒绝
4. **可以取消**：点击"取消"按钮会恢复审批按钮

## 🚀 部署步骤

### 第 1 步：执行数据库迁移

**时间：5 分钟**

1. 打开 [Supabase 控制台](https://supabase.com/dashboard)
2. 选择项目
3. 点击 "SQL Editor" → "New Query"
4. 复制 `database/add_emperor_sarcasm_logs_simple.sql`
5. 粘贴到编辑器
6. 点击 "Run"

**验证：**
```sql
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('emperor_sarcasm_logs', 'minister_log_progress');
```

### 第 2 步：重启后端

**时间：2 分钟**

```bash
cd backend
npm run dev
```

**验证：**
```bash
curl http://localhost:3001/api/logs/unviewed/test-court/test-user
```

应该返回 200 状态码。

### 第 3 步：测试 API

**时间：5 分钟**

```bash
# 创建测试日志
curl -X POST http://localhost:3001/api/logs/create \
  -H "Content-Type: application/json" \
  -d '{
    "courtId": "test-court",
    "userId": "test-user",
    "logType": "task_assigned",
    "logMessage": "[14:30] 皇帝让大臣A去唱歌，大臣A连屁都不敢放，乖乖接受了"
  }'

# 获取未查看日志
curl http://localhost:3001/api/logs/unviewed/test-court/test-user

# 标记为已查看
curl -X POST http://localhost:3001/api/logs/mark-viewed \
  -H "Content-Type: application/json" \
  -d '{
    "courtId": "test-court",
    "userId": "test-user",
    "logIds": ["log-id"]
  }'
```

## 📋 集成清单

### 游戏逻辑集成

需要在以下地方添加日志调用：

#### 1. 任务分配时
**文件：** `backend/src/routes/tasks.ts`

```typescript
import { logTaskAssignment } from '../services/sarcasm-logs.service';

// 在分配任务的地方添加
await logTaskAssignment(
  courtId,
  ministerId,
  ministerName,
  taskName,
  taskId
);
```

#### 2. 任务完成时
**文件：** `backend/src/routes/tasks.ts`

```typescript
import { logTaskSuccess, logTaskFailure } from '../services/sarcasm-logs.service';

// 任务成功
if (score >= 60) {
  await logTaskSuccess(courtId, ministerId, ministerName, taskName, taskId, score);
}

// 任务失败
if (score < 60) {
  await logTaskFailure(courtId, ministerId, ministerName, taskName, taskId, score);
}
```

#### 3. 排名变化时
**文件：** `backend/src/services/grudge.service.ts` 或相关排名计算文件

```typescript
import { logRankingChange } from '../services/sarcasm-logs.service';

// 在排名更新后添加
if (newRank !== oldRank) {
  await logRankingChange(
    courtId,
    ministerId,
    ministerName,
    oldRank,
    newRank,
    oldScore,
    newScore
  );
}
```

#### 4. 皇帝嘲笑时
**文件：** `backend/src/routes/courts.ts` 或相关皇帝操作文件

```typescript
import { logEmperorMocking } from '../services/sarcasm-logs.service';

// 在皇帝发送嘲笑消息时添加
await logEmperorMocking(
  courtId,
  ministerId,
  ministerName,
  mockingMessage
);
```

#### 5. 惩罚时
**文件：** `backend/src/routes/courts.ts` 或相关惩罚文件

```typescript
import { logPunishment } from '../services/sarcasm-logs.service';

// 在执行惩罚时添加
await logPunishment(
  courtId,
  ministerId,
  ministerName,
  'virtual'  // 或 'public', 'salary'
);
```

#### 6. 奖励时
**文件：** `backend/src/routes/courts.ts` 或相关奖励文件

```typescript
import { logReward } from '../services/sarcasm-logs.service';

// 在执行奖励时添加
await logReward(
  courtId,
  ministerId,
  ministerName,
  'salary',  // 或 'promotion', 'reward'

```

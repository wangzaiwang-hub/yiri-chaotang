# 修复404错误和无限循环问题

## 问题总结

1. **无限循环错误** ✅ 已修复
   - 位置: `frontend/src/pages/Home.tsx` line 564-569
   - 原因: useEffect 依赖数组包含 `currentCourtId`，但在 effect 内部调用 `setCurrentCourtId`，导致无限循环
   - 修复: 从依赖数组中移除 `currentCourtId`

2. **404 错误** - 需要重新部署后端
   - 错误: `/api/territory/{courtId}` 和 `/api/territory/{courtId}/happiness` 返回 404
   - 原因: Railway 上的后端代码还没有包含新的 territory 路由
   - 解决方案: 重新部署后端到 Railway

## 已完成的修复

### 1. 修复无限循环

```typescript
// 修改前
useEffect(() => {
  if (currentCourt?.id && currentCourt.id !== currentCourtId) {
    localStorage.setItem('currentCourtId', currentCourt.id);
    setCurrentCourtId(currentCourt.id);
  }
}, [currentCourt?.id, currentCourtId]); // ❌ currentCourtId 导致无限循环

// 修改后
useEffect(() => {
  if (currentCourt?.id && currentCourt.id !== currentCourtId) {
    localStorage.setItem('currentCourtId', currentCourt.id);
    setCurrentCourtId(currentCourt.id);
  }
}, [currentCourt?.id]); // ✅ 移除 currentCourtId
```

## 技术细节

### Territory API 路由

后端提供以下 API:

1. `GET /api/territory/:courtId` - 获取疆土拓展数据
2. `POST /api/territory/:courtId/happiness` - 更新幸福值
3. `POST /api/territory/:courtId/unlock-scene` - 解锁新场景
4. `POST /api/territory/:courtId/switch-scene` - 切换当前场景
5. `GET /api/territory/scenes/all` - 获取所有场景配置

所有路由都需要 `user_id` 参数进行身份验证。

## 🚀 快速开始

### 步骤 1：执行数据库迁移

1. 访问 [Supabase 控制台](https://supabase.com/dashboard)
2. 选择你的项目
3. 点击 "SQL Editor" → "New Query"
4. 复制 `database/add_emperor_sarcasm_logs_simple.sql` 中的所有代码
5. 粘贴到 SQL 编辑器，点击 "Run"

### 步骤 2：验证迁移

在 SQL 编辑器执行：

```sql
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('emperor_sarcasm_logs', 'minister_log_progress');
```

应该看到两个表。

### 步骤 3：重启后端

```bash
cd backend
npm run dev
```

后端会自动加载新的路由。

## 🔌 API 调用示例

### 获取未查看日志

```bash
curl http://localhost:3001/api/logs/unviewed/court-id/user-id
```

**响应：**
```json
{
  "success": true,
  "unviewedLogs": [
    {
      "id": "log-id",
      "log_type": "task_assigned",
      "log_message": "[14:30] 皇帝让大臣A去唱歌，大臣A连屁都不敢放，乖乖接受了",
      "related_data": {"taskId": "task-1", "taskName": "唱歌"},
      "created_at": "2026-04-07T14:30:00Z"
    }
  ],
  "unviewedCount": 1
}
```

### 标记日志为已查看

```bash
curl -X POST http://localhost:3001/api/logs/mark-viewed \
  -H "Content-Type: application/json" \
  -d '{
    "courtId": "court-id",
    "userId": "user-id",
    "logIds": ["log-id-1", "log-id-2"]
  }'
```

**响应：**
```json
{
  "success": true,
  "message": "标记日志为已查看成功"
}
```

## 📊 数据库结构

| id | UUID | 日志唯一标识 |

## 🔧 故障排查

### Q: 迁移失败？

A: 确保：
1. 使用的是 `SUPABASE_SERVICE_KEY` 而不是 `SUPABASE_ANON_KEY`
2. 在 Supabase 控制台的 SQL Editor 中执行
3. 没有语法错误

### Q: API 返回 404？

A: 确保：
1. 后端已重启
2. 路由已正确注册
3. 使用正确的 URL 路径

### Q: 日志没有创建？

A: 检查：
1. 数据库连接是否正常
2. 参数是否正确
3. 查看后端日志输出

# 召唤分身功能 - 开发完成

## 功能概述

已成功实现"召唤分身"功能，允许皇帝在没有真实玩家在线时，召唤其他用户的 SecondMe 虚拟分身加入朝堂，实现单人游戏体验。

## 完成内容

### 后端实现 ✅

**文件：`backend/src/routes/courts.ts`**

1. **获取可用人机列表 API**
   - 路由：`GET /api/courts/:id/available-bots`
   - 功能：返回不在当前朝堂的所有用户（最多 20 个）
   - 去重：自动排除当前朝堂成员

2. **召唤分身 API**
   - 路由：`POST /api/courts/:id/summon-bot`
   - 功能：将指定用户添加为朝堂成员
   - 随机分配：自动随机分配到三省六部之一
   - 默认设置：角色为 minister，怨气值为 0，性别为 male

### 前端实现 ✅

**文件：`frontend/src/services/api.ts`**

添加了两个新的 API 方法：
```typescript
getAvailableBots: (courtId: string, currentUserId: string) => ...
summonBot: (courtId: string, userId: string) => ...
```

**文件：`frontend/src/pages/Home.tsx`**

1. **新增状态**
   - `showSummonModal`：控制弹窗显示
   - `availableBots`：存储可用人机列表
   - `isSummoning`：召唤进行中状态

2. **新增函数**
   - `handleOpenSummonModal()`：打开弹窗并获取人机列表
   - `handleSummonBot()`：召唤指定分身

3. **UI 组件**
   - 左上角新增"召唤分身"按钮（紫色边框）
   - 召唤分身弹窗（3 列网格布局）
   - 分身卡片（显示头像、昵称）

## UI 设计

### 按钮样式
- 位置：左上角第二个按钮（在"招贤纳士"下方）
- 颜色：紫色边框 + 紫色文字
- 文字："召唤分身"
- 样式：白色半透明背景，圆角，阴影

### 弹窗样式
- 背景：紫色到粉色渐变
- 标题："召唤分身"（金色渐变文字）
- 布局：3 列网格，最多显示 20 个分身
- 卡片：白色背景，紫色边框，悬停放大

## 使用流程

1. 皇帝点击左上角"召唤分身"按钮
2. 弹出分身列表，显示所有可召唤的用户
3. 点击任意分身卡片
4. 系统自动召唤该分身，随机分配部门
5. 显示 Toast 提示"已召唤 XX 分身入朝"
6. 页面自动刷新，分身出现在朝堂中

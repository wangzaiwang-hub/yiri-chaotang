### 1. 提交代码到 Git

```bash

git add backend/test-log-integration.js
git add docs/日志系统集成完成.md
git add docs/日志系统-立即部署.md
```

### 2. Railway 会自动部署

- Railway 检测到代码变更后会自动重新部署
- 等待部署完成（通常 2-3 分钟）
- 查看 Railway 日志确认部署成功

## 故障排查

### 如果日志没有创建

1. **检查后端日志**
   ```
   在 Railway 控制台搜索 "📝 创建"
   ```

2. **检查数据库**
   ```sql
   SELECT * FROM emperor_sarcasm_logs 
   ORDER BY created_at DESC 
   LIMIT 10;
   ```

3. **检查函数是否存在**
   ```sql
   SELECT proname FROM pg_proc 
   WHERE proname LIKE '%sarcasm%';
   ```

### 如果前端不显示

1. **检查浏览器控制台**
   - 打开开发者工具
   - 查看 Console 标签
   - 搜索 "日志" 或 "log"

2. **检查 API 响应**
   ```bash
   curl https://backend-production-a216.up.railway.app/api/logs/unviewed/COURT_ID/USER_ID
   ```

3. **检查角色判断**
   - 确保是大臣端，不是皇帝端
   - 皇帝端不会显示日志弹窗

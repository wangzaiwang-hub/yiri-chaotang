## 数据流

### 1. 创建日志流程

```
游戏逻辑
  ↓
调用 logTaskAssignment()
  ↓
生成讽刺消息
  ↓
调用 createSarcasmLog()
  ↓
RPC 调用 create_sarcasm_log()
  ↓
插入 emperor_sarcasm_logs 表
  ↓
触发 update_unviewed_count_trigger
  ↓
更新 minister_log_progress 表
```

### 2. 显示日志流程

```
大臣登录
  ↓
Home.tsx useEffect 触发
  ↓
调用 logsAPI.getUnviewed()
  ↓
GET /api/logs/unviewed/:courtId/:userId
  ↓
调用 getUnviewedLogs()
  ↓
RPC 调用 get_unviewed_logs()
  ↓
查询 emperor_sarcasm_logs 表
  ↓
返回未查看日志列表
  ↓
显示 LogModal
  ↓
大臣查看所有日志
  ↓
点击确认
  ↓
调用 logsAPI.markAsViewed()
  ↓
POST /api/logs/mark-viewed
  ↓
调用 markLogsAsViewed()
  ↓
RPC 调用 mark_logs_as_viewed()
  ↓
更新 emperor_sarcasm_logs 表 (viewed_by_minister = true)
  ↓
触发 update_unviewed_count_trigger
  ↓
更新 minister_log_progress 表
  ↓
关闭 LogModal
  ↓
继续游戏
```

## 讽刺消息生成流程

```
游戏事件发生
  ↓
调用专用日志函数
  ├─ logTaskAssignment()
  ├─ logTaskFailure()
  ├─ logTaskSuccess()
  ├─ logRankingChange()
  ├─ logEmperorMocking()
  ├─ logPunishment()
  └─ logReward()
  ↓
从消息数组中随机选择
  ↓
添加时间戳
  ↓
调用 createSarcasmLog()
  ↓
保存到数据库
  ↓
大臣下次登录时看到
```

## 状态转换

```
┌─────────────────┐
│  未查看日志     │
│  (viewed=false) │
└────────┬────────┘
         │
         │ 大臣查看
         ↓
┌─────────────────┐
│  已查看日志     │
│  (viewed=true)  │
└─────────────────┘
```

## 组件交互

```
Home.tsx
  ├─ 检查未查看日志
  │   └─ logsAPI.getUnviewed()
  │
  ├─ 显示 LogModal
  │   ├─ 显示日志内容
  │   ├─ 导航按钮
  │   └─ 确认按钮
  │
  └─ 标记日志为已查看
      └─ logsAPI.markAsViewed()
```

## 性能优化

```
查询优化
  ├─ 索引: court_id + user_id
  ├─ 索引: viewed_by_minister
  └─ 分页: limit + offset

缓存策略
  ├─ 前端: 本地状态管理
  ├─ 后端: 数据库查询缓存
  └─ 数据库: 索引加速

并发处理
  ├─ 事务: 原子性操作
  ├─ 锁: 防止竞态条件
  └─ 触发器: 自动更新
```

## 错误处理

```
前端错误
  ├─ 网络错误 → 显示提示
  ├─ 参数错误 → 验证输入
  └─ 服务错误 → 显示错误信息

后端错误
  ├─ 参数验证 → 返回 400
  ├─ 数据库错误 → 返回 500
  └─ 业务逻辑错误 → 返回相应状态码

数据库错误
  ├─ 约束违反 → 事务回滚
  ├─ 连接错误 → 重试机制
  └─ 超时 → 返回错误
```

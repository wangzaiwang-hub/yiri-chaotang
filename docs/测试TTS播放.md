# 测试 TTS 语音播放

## 快速测试步骤

### 1. 打开浏览器开发者工具
按 `F12` 或 `Cmd+Option+I` (Mac)

### 2. 切换到 Console 标签
观察日志输出

### 3. 在游戏中触发语音
- 作为皇帝，发布任务给大臣
- 等待大臣回复
- 观察气泡右上角

### 4. 查看结果

#### 情况 A：自动播放成功 ✅
- 看到绿色"播放中"指示器（带脉冲动画）
- 听到语音
- Console 显示：
  ```
  🔊 开始生成 TTS 语音...
  🔊 音频 URL: https://...
  🔊 播放命令已发送
  🔊 音频开始播放
  ```

#### 情况 B：自动播放被阻止 ⚠️
- 看到蓝色"点击播放"按钮（带跳动动画）
- 没有声音
- Console 显示：
  ```
  🔊 播放被阻止: NotAllowedError
  🔊 浏览器阻止了自动播放，显示手动播放按钮
  ```
- **解决方法**：点击蓝色"点击播放"按钮

#### 情况 C：音频加载失败 ❌
- 显示错误提示
- Console 显示：
  ```
  🔊 音频播放失败: ...
  🔊 音频错误详情: { code: 4, ... }
  ```
- **可能原因**：
  - 网络问题
  - 音频 URL 失效
  - CORS 问题

#### 情况 D：TTS 生成失败 ❌
- 显示"语音生成失败"提示
- Console 显示：
  ```
  🔊 TTS 生成失败: ...
  ```
- **可能原因**：
  - 后端服务未运行
  - 用户未设置语音
  - API 调用失败

## 调试技巧

### 1. 复制音频 URL 测试
从 Console 复制 `🔊 音频 URL:` 后面的链接，在新标签页打开：
- 能播放 → 浏览器策略问题，点击"点击播放"按钮
- 不能播放 → URL 问题，检查 SecondMe 设置

### 2. 检查网络请求
开发者工具 → Network 标签 → 筛选 XHR：
- 查看 `/api/tts/generate` 请求
- 检查响应状态码和返回数据

### 3. 检查音频元素
Console 中输入：
```javascript
// 查看当前音频状态
console.log(document.querySelector('audio'))
```

### 4. 手动测试 TTS API
```bash
cd backend
node test-tts-direct.js
```

## 允许自动播放（推荐）

### Chrome
1. 点击地址栏左侧的锁图标
2. 网站设置
3. 声音 → 允许

### Safari
1. Safari 菜单 → 偏好设置
2. 网站 → 自动播放
3. 当前网站 → 允许所有自动播放

### Firefox
1. 点击地址栏左侧的图标
2. 权限 → 自动播放
3. 允许音频和视频

## 预期行为

### 正常流程
```
1. 大臣回复任务
2. 气泡显示打字机效果
3. 打字完成后 3 秒
4. 气泡淡出
5. 显示"播放中"指示器（或"点击播放"按钮）
6. 播放语音
7. 播放完成，指示器消失
8. 显示"结准驳"按钮
```

### 时间线
```
0s   - 大臣回复到达
0-2s - 打字机效果
2-5s - 气泡停留
5-6s - 气泡淡出
6s   - 开始播放语音（或显示播放按钮）
6-10s - 语音播放中
10s  - 显示审批按钮
```

## 常见问题

### Q: 为什么有时能自动播放，有时不能？
A: 取决于浏览器是否认为用户有足够的交互。建议在浏览器设置中允许自动播放。

### Q: 点击"点击播放"按钮后还是没声音？
A: 
1. 检查系统音量
2. 检查浏览器标签页是否静音
3. 查看 Console 是否有新的错误
4. 复制音频 URL 在新标签页测试

### Q: 每次都要点击播放按钮吗？
A: 不是。允许自动播放后，之后就能自动播放了。

### Q: 如何确认用户已设置语音？
A: 
1. 登录 SecondMe 应用
2. 进入个人设置
3. 查看是否有语音设置选项
4. 如果没有，需要先设置语音克隆或选择预设语音

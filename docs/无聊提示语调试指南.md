# 无聊提示语功能调试指南

## 功能说明
当朝堂里没有大臣时，皇帝的 SecondMe 数字分身会每 5-10 秒随机生成一句无聊的话，使用打字机效果逐字显示。

## 调试步骤

### 1. 清除浏览器缓存
- Mac: `Cmd + Shift + R`
- Windows: `Ctrl + Shift + R`
- 或者在开发者工具中右键刷新按钮，选择"清空缓存并硬性重新加载"

### 2. 查看浏览器控制台日志
打开浏览器开发者工具（F12），查看 Console 标签，应该能看到：

```
🤴 皇帝无聊模式启动，准备调用 SecondMe 生成提示语
📡 正在调用 SecondMe API 生成无聊提示语...
✅ SecondMe 返回消息: xxx
```

如果看到：
```
⏸️ 无聊模式未启动: { ministers: 0, isEmperor: true, hasCourt: true, hasUserId: true }
```
说明某个条件不满足。

### 3. 查看后端终端日志
后端应该输出：

```
🤴 收到无聊提示语请求, emperor_id: xxx
✅ 找到皇帝 token，准备调用 SecondMe
[SecondMe 流式响应内容]
💬 SecondMe 生成的消息: xxx
```

### 4. 检查网络请求
在开发者工具的 Network 标签：
- 查找 `bored-message` 请求
- 检查请求参数：`{ emperor_id: "xxx" }`
- 检查响应：`{ code: 0, data: { message: "xxx" } }`

## 功能参数

### 打字速度
- 当前设置：150ms/字
- 修改位置：`frontend/src/pages/Home.tsx` 第 395 行
- 可调整范围：50-300ms

### 显示时间
- 打字完成后显示 3 秒
- 修改位置：`frontend/src/pages/Home.tsx` 第 401 行

### 触发频率
- 首次：页面加载后 2 秒
- 之后：每 5-10 秒随机触发一次
- 修改位置：`frontend/src/pages/Home.tsx` 第 410-415 行

## 常见问题

### Q: 还是显示固定的几句话
A: 
1. 清除浏览器缓存
2. 检查前端是否重新编译（Vite 应该自动重新加载）
3. 查看控制台是否有 API 调用日志

### Q: 打字太快/太慢
A: 修改 `frontend/src/pages/Home.tsx` 第 395 行的 `150` 值（单位：毫秒）

### Q: API 调用失败
A: 
1. 检查后端是否运行
2. 检查皇帝是否有 SecondMe token
3. 查看后端终端的错误日志

### Q: 终端显示 SecondMe 响应但前端没显示
A: 
1. 检查浏览器控制台的网络请求
2. 确认响应格式是否正确：`{ code: 0, data: { message: "xxx" } }`
3. 检查前端是否正确解析响应：`response.data?.data?.message`

## 测试脚本

可以使用测试脚本直接测试后端 API：

```bash
cd backend
node test-bored-message.js
```

记得先修改脚本中的 `YOUR_EMPEROR_USER_ID` 为实际的用户 ID。

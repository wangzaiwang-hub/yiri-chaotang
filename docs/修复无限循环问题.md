# 修复 React 无限循环问题

## 问题描述

前端出现 React 错误：
```
Warning: Maximum update depth exceeded. This can happen when a component calls setState inside useEffect, but useEffect either doesn't have a dependency array, or one of the dependencies changes on every render.
```

## 问题原因

### 问题 1：currentCourtId 无限循环

**位置**：`frontend/src/pages/Home.tsx` 第 500-507 行

**原因**：
```typescript
useEffect(() => {
  if (currentCourt?.id) {
    localStorage.setItem('currentCourtId', currentCourt.id);
    setCurrentCourtId(currentCourt.id); // 这里会触发状态更新
  }
}, [currentCourt?.id]); // 但依赖项中没有 currentCourtId
```

当 `setCurrentCourtId` 被调用时，可能会影响 `currentCourt` 的计算，导致 `currentCourt?.id` 变化，从而再次触发 useEffect，形成无限循环。

**修复**：
```typescript
useEffect(() => {
  if (currentCourt?.id && currentCourt.id !== currentCourtId) {
    localStorage.setItem('currentCourtId', currentCourt.id);
    setCurrentCourtId(currentCourt.id);
  }
}, [currentCourt?.id, currentCourtId]); // 添加 currentCourtId 依赖，并检查是否真的需要更新
```

### 问题 2：showGenderModal 依赖缺失

**位置**：`frontend/src/pages/Home.tsx` 第 970-974 行

**原因**：
```typescript
useEffect(() => {
  if (needsGenderSelection && currentCourt?.id && !showGenderModal) {
    setShowGenderModal(true); // 修改了 showGenderModal
  }
}, [needsGenderSelection, currentCourt?.id]); // 但依赖项中没有 showGenderModal
```

虽然这个不会直接导致无限循环（因为有 `!showGenderModal` 的检查），但缺少依赖项会导致 React 警告。

**修复**：
```typescript
useEffect(() => {
  if (needsGenderSelection && currentCourt?.id && !showGenderModal) {
    setShowGenderModal(true);
  }
}, [needsGenderSelection, currentCourt?.id, showGenderModal]); // 添加 showGenderModal 依赖
```

## 修复后的效果

- ✅ 不再出现 "Maximum update depth exceeded" 错误
- ✅ 不再有 React 依赖项警告
- ✅ 组件正常渲染，不会无限重新渲染
- ✅ localStorage 正确保存 currentCourtId
- ✅ 性别选择弹窗正常显示

修复后，打开浏览器控制台，应该：

2. 不再看到黄色的 React 依赖项警告

## 最佳实践

### useEffect 依赖项规则

1. **包含所有使用的变量**：useEffect 中使用的所有 props、state 和函数都应该在依赖数组中
2. **避免无限循环**：如果在 useEffect 中调用 setState，确保有条件判断避免无限循环
3. **使用 ESLint 规则**：启用 `react-hooks/exhaustive-deps` 规则自动检测依赖项问题

### 示例：正确的 useEffect 模式

```typescript
// ✅ 好的做法：有条件判断，避免无限循环
useEffect(() => {
  if (value !== prevValue) {
    setValue(newValue);
  }
}, [value, prevValue]);

// ❌ 坏的做法：无条件 setState，会导致无限循环
useEffect(() => {
  setValue(newValue);
}, [value]);

// ✅ 好的做法：包含所有依赖项
useEffect(() => {
  if (condition && !flag) {
    setFlag(true);
  }
}, [condition, flag]);

// ❌ 坏的做法：缺少依赖项
useEffect(() => {
  if (condition && !flag) {
    setFlag(true);
  }
}, [condition]); // 缺少 flag
```

## 无能指数计算

### 无能指数 = 失败率 × 任务难度 × 时间系数

```typescript
interface IncompetenceScore {
  failureRate: number;           // 失败率（0-100）
  taskDifficulty: number;        // 任务难度（1-10）
  timeCoefficient: number;       // 时间系数（越久越无能）
  totalScore: number;            // 总无能指数
  rank: number;                  // 排名
}
```

### 具体计算

```typescript
const incompetenceScore = {
  // 失败率（完成失败的任务数 / 总任务数）
  failureRate: failedTasks / totalTasks * 100,
  
  // 任务难度（平均难度）
  taskDifficulty: averageTaskDifficulty,
  
  // 时间系数（在朝堂中的天数）
  timeCoefficient: daysInCourt / 10,
  
  // 总分 = 失败率 × 难度 × 时间
  totalScore: (failureRate * taskDifficulty * timeCoefficient) / 100
};

// 例子：
// 失败率 50%，平均难度 5，在朝堂 10 天
// 无能指数 = (50 × 5 × 1) / 100 = 2.5
```

## 排行榜的更新机制

### 实时更新

```typescript
// 每次任务完成/失败时更新
const updateIncompetenceScore = async (ministerId: string) => {
  // 1. 计算失败率
  const failureRate = failedTasks / totalTasks * 100;
  
  // 2. 计算平均难度
  const avgDifficulty = totalDifficulty / totalTasks;
  
  // 3. 计算时间系数
  const timeCoeff = daysInCourt / 10;
  
  // 4. 计算总分
  const score = (failureRate * avgDifficulty * timeCoeff) / 100;
  
  // 5. 更新排行榜
  await updateRanking(ministerId, score);
  
  // 6. 如果排名变化，通知大臣
  if (newRank !== oldRank) {
    notifyMinister(`你的无能排名从第 ${oldRank} 升到第 ${newRank}`);
  }
};
```

## 数据库设计

```sql
-- 无能指数表
CREATE TABLE incompetence_scores (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  court_id UUID REFERENCES courts(id),
  user_id UUID REFERENCES users(id),
  
  -- 计算数据
  total_tasks INTEGER DEFAULT 0,
  failed_tasks INTEGER DEFAULT 0,
  failure_rate DECIMAL(5, 2) DEFAULT 0,
  avg_difficulty DECIMAL(3, 1) DEFAULT 0,
  days_in_court INTEGER DEFAULT 0,
  
  -- 最终分数
  incompetence_score DECIMAL(5, 2) DEFAULT 0,
  rank INTEGER,
  level TEXT,  -- S, A, B, C, D, E
  
  -- 历史记录
  last_failed_task TEXT,
  failed_tasks_list TEXT[],
  
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 排行榜历史
CREATE TABLE incompetence_history (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  court_id UUID REFERENCES courts(id),
  user_id UUID REFERENCES users(id),
  score DECIMAL(5, 2),
  rank INTEGER,
  recorded_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 这个系统的气人之处

1. **公开羞辱**
   - 排行榜公开显示
   - 所有人都能看到你的排名
   - 无法隐瞒

2. **无法反驳**
   - 数据是客观的
   - 失败的任务都记录在案
   - 无法狡辩

3. **持续提醒**
   - 每次登录都看到排行榜
   - 每次任务失败都更新排名
   - 无法逃避

4. **越来越无能**
   - 无能指数不断上升
   - 排名越来越靠前
   - 看着自己变得越来越无能

5. **皇帝的嘲笑**
   - 皇帝可以分享排行榜
   - 皇帝可以发送嘲笑消息
   - 大臣无法反驳

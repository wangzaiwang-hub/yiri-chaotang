# 皇帝毒舌日志系统 - 快速参考

## 🚀 快速开始

### 后端调用

```typescript
// 导入
import { 
  logTaskAssignment, 
  logTaskFailure, 
  logTaskSuccess,
  logRankingChange,
  logEmperorMocking,
  logPunishment,
  logReward
} from '../services/sarcasm-logs.service';

// 任务分配
await logTaskAssignment(courtId, ministerId, ministerName, taskName, taskId);

// 任务失败
await logTaskFailure(courtId, ministerId, ministerName, taskName, taskId, score);

// 任务成功
await logTaskSuccess(courtId, ministerId, ministerName, taskName, taskId, score);

// 排名变化
await logRankingChange(courtId, ministerId, ministerName, oldRank, newRank, oldScore, newScore);

// 皇帝嘲笑
await logEmperorMocking(courtId, ministerId, ministerName, mockingMessage);

// 惩罚
await logPunishment(courtId, ministerId, ministerName, punishmentType);

// 奖励
await logReward(courtId, ministerId, ministerName, rewardType, rewardAmount);
```


## 📊 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/logs/create` | POST | 创建日志 |
| `/api/logs/unviewed/:courtId/:userId` | GET | 获取未查看日志 |
| `/api/logs/history/:courtId/:userId` | GET | 获取日志历史 |
| `/api/logs/mark-viewed` | POST | 标记日志为已查看 |
| `/api/logs/progress/:courtId/:userId` | GET | 获取日志进度 |
| `/api/logs/unviewed-count/:courtId/:userId` | GET | 获取未查看日志数量 |

## 📝 参数说明

### 任务分配
- `courtId`: 朝堂ID
- `ministerId`: 大臣ID
- `ministerName`: 大臣名字
- `taskName`: 任务名称
- `taskId`: 任务ID

### 任务失败/成功
- `courtId`: 朝堂ID
- `ministerId`: 大臣ID
- `ministerName`: 大臣名字
- `taskName`: 任务名称
- `taskId`: 任务ID
- `score`: 分数

### 排名变化
- `courtId`: 朝堂ID
- `ministerId`: 大臣ID
- `ministerName`: 大臣名字
- `oldRank`: 旧排名
- `newRank`: 新排名
- `oldScore`: 旧无能指数
- `newScore`: 新无能指数

### 皇帝嘲笑
- `courtId`: 朝堂ID
- `ministerId`: 大臣ID
- `ministerName`: 大臣名字
- `mockingMessage`: 嘲笑内容

### 惩罚
- `courtId`: 朝堂ID
- `ministerId`: 大臣ID
- `ministerName`: 大臣名字
- `punishmentType`: 惩罚类型 (`virtual` | `public` | `salary`)

### 奖励
- `courtId`: 朝堂ID
- `ministerId`: 大臣ID
- `ministerName`: 大臣名字
- `rewardType`: 奖励类型 (`salary` | `promotion` | `reward`)
- `rewardAmount`: 奖励数额（可选）

## 🔍 常见问题

**Q: 日志什么时候显示？**
A: 大臣登录游戏时，如果有未查看的日志，会自动显示 LogModal。

**Q: 大臣可以跳过日志吗？**
A: 不可以。必须查看所有日志才能点击确认按钮。

**Q: 如何自定义讽刺消息？**
A: 编辑 `backend/src/services/sarcasm-logs.service.ts` 中的消息数组。

**Q: 日志会永久保存吗？**
A: 是的。所有日志都会保存在数据库中。

# 你的部署步骤

✅ 代码已推送到：https://github.com/wangzaiwang-hub/yiri-chaotang

## 下一步：部署到 Railway 和 Vercel

### 第一步：部署后端到 Railway（5分钟）

1. **访问 Railway**
   - 打开 https://railway.app
   - 使用 GitHub 账号登录

2. **创建新项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择 `wangzaiwang-hub/yiri-chaotang`

3. **配置项目**
   - 点击项目进入设置
   - 在 "Settings" 标签中：
     - Root Directory: `backend`
   
4. **添加环境变量**
   - 点击 "Variables" 标签
   - 点击 "New Variable" 添加以下变量：

```
NODE_ENV=production
PORT=3001

# Supabase（从你的 backend/.env 文件复制）
SUPABASE_URL=你的Supabase URL
SUPABASE_ANON_KEY=你的Supabase匿名密钥
SUPABASE_SERVICE_ROLE_KEY=你的Supabase服务密钥

# SecondMe（从你的 backend/.env 文件复制）
SECONDME_CLIENT_ID=你的SecondMe客户端ID
SECONDME_CLIENT_SECRET=你的SecondMe客户端密钥
SECONDME_API_KEY=你的SecondMe API密钥

# 应用URL（先填临时值，部署后更新）
BASE_URL=https://temporary.up.railway.app
FRONTEND_URL=https://temporary.vercel.app
```

5. **部署**
   - Railway 会自动开始部署
   - 等待几分钟，直到显示 "Success"
   - 点击 "Settings" → "Networking" → "Generate Domain"
   - 记录生成的域名（例如：`https://yiri-chaotang-production.up.railway.app`）

6. **更新环境变量**
   - 回到 "Variables" 标签
   - 更新 `BASE_URL` 为刚才生成的 Railway 域名
   - 保存后会自动重新部署

---

### 第二步：部署前端到 Vercel（3分钟）

1. **访问 Vercel**
   - 打开 https://vercel.com/new
   - 使用 GitHub 账号登录

2. **导入项目**
   - 点击 "Import Git Repository"
   - 选择 `wangzaiwang-hub/yiri-chaotang`
   - 点击 "Import"

3. **配置项目**
   - Project Name: `yiri-chaotang`（或自定义）
   - Framework Preset: `Vite`
   - Root Directory: `frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`

4. **添加环境变量**
   - 展开 "Environment Variables"
   - 添加：
     ```
     Name: VITE_API_URL
     Value: https://你的Railway域名.up.railway.app
     ```
   - 例如：`https://yiri-chaotang-production.up.railway.app`

5. **部署**
   - 点击 "Deploy"
   - 等待几分钟，直到显示 "Congratulations!"
   - 记录 Vercel 提供的域名（例如：`https://yiri-chaotang.vercel.app`）

---

### 第三步：更新配置（2分钟）

#### 3.1 更新 Railway 的 FRONTEND_URL

1. 回到 Railway 项目
2. 进入 "Variables" 标签
3. 更新 `FRONTEND_URL` 为 Vercel 域名
4. 保存（会自动重新部署）

#### 3.2 更新 SecondMe 回调 URL

1. 访问 SecondMe 开发者平台：https://develop.second-me.cn/integrations/list
2. 找到你的应用
3. 更新回调 URL 为：
   ```
   https://你的Railway域名.up.railway.app/api/auth/callback
   ```
4. 保存

#### 3.3 更新 Supabase 配置

1. 访问 Supabase 项目：https://supabase.com/dashboard
2. 进入你的项目
3. 点击 "Authentication" → "URL Configuration"
4. 更新：
   - Site URL: `https://你的Vercel域名.vercel.app`
   - Redirect URLs: 添加 `https://你的Vercel域名.vercel.app/**`
5. 保存

---

### 第四步：测试（1分钟）

1. **访问前端**
   - 打开你的 Vercel 域名
   - 应该能看到登录页面

2. **测试登录**
   - 点击"使用 SecondMe 登录"
   - 完成授权
   - 应该能成功登录并看到主页

3. **测试功能**
   - 创建朝堂
   - 邀请成员
   - 发布任务
   - 测试准奏/驳回

## 遇到问题？

### 后端部署失败
- 检查 Railway 部署日志
- 确认环境变量是否正确配置
- 确认 Supabase 数据库连接正常

### 前端无法访问后端
- 检查 `VITE_API_URL` 是否正确
- 检查后端 CORS 配置
- 查看浏览器控制台的网络请求

### 登录失败
- 检查 SecondMe 回调 URL 是否正确
- 检查 `BASE_URL` 和 `FRONTEND_URL` 是否正确
- 检查 SecondMe API 密钥是否有效

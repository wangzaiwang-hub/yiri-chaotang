# 皇帝毒舌日志系统 - 大臣端触发说明

## 触发逻辑

日志弹窗**只在大臣端登录时触发**，皇帝端不会看到这些日志。

## 实现细节

### 代码位置
`frontend/src/pages/Home.tsx`

### 判断逻辑

```typescript
const ranking = rankingRes?.data.data || [];
const emperor = ranking.find((m: any) => m.role === 'emperor');
const ministers = ranking.filter((m: any) => m.role === 'minister');
const isEmperor = emperor?.user_id === user?.id;

// 检查未查看的日志（只在大臣端触发）
useEffect(() => {
  const checkUnviewedLogs = async () => {
    // 只有大臣才检查日志
    if (!currentCourt?.id || !user?.id || isEmperor) return;
    
    try {
      const response = await logsAPI.getUnviewed(currentCourt.id, user.id);
      if (response.data.unviewedLogs && response.data.unviewedLogs.length > 0) {
        setUnviewedLogs(response.data.unviewedLogs);
        setShowLogModal(true);
      }
    } catch (error) {
      console.error('获取未查看日志失败:', error);
    }
  };
  
  checkUnviewedLogs();
}, [currentCourt?.id, user?.id, isEmperor]);
```

### 关键点

1. **角色判断**: 通过 `isEmperor` 判断当前用户是否为皇帝
2. **提前返回**: 如果是皇帝（`isEmperor === true`），直接返回，不检查日志
3. **依赖项**: useEffect 依赖于 `[currentCourt?.id, user?.id, isEmperor]`，确保角色变化时重新检查

## 用户体验

### 大臣端
1. 大臣登录游戏
2. 系统自动检查是否有未查看的日志
3. 如果有日志，显示 LogModal
4. 大臣必须查看所有日志才能继续游戏
5. 点击确认后，标记日志为已查看

### 皇帝端
1. 皇帝登录游戏
2. 系统不检查日志
3. 不显示 LogModal
4. 直接进入游戏

## 日志创建

日志由皇帝的操作触发，记录到数据库中，但只有大臣登录时才会看到这些日志。

### 创建日志的操作
- 任务分配
- 任务失败
- 任务成功
- 排名变化
- 皇帝嘲笑
- 惩罚执行
- 奖励发放

## 数据流

```
皇帝操作
  ↓
创建日志（记录到数据库）
  ↓
日志保存（user_id = 大臣ID）
  ↓
大臣登录
  ↓
检查角色（isEmperor === false）
  ↓
获取未查看日志
  ↓
显示 LogModal
  ↓
大臣查看并确认
  ↓
标记日志为已查看
```

## 注意事项

1. **角色判断时机**: 必须在 `ranking` 数据加载完成后才能判断角色
2. **useEffect 位置**: 日志检查的 useEffect 必须放在 `isEmperor` 定义之后
3. **依赖项**: 必须包含 `isEmperor` 作为依赖项，确保角色变化时重新检查

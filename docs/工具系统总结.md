## 🎯 工作流程

```
皇帝下旨
    ↓
大臣虚拟人分析任务
    ↓
决定是否使用工具
    ↓
[如果需要工具]
    ↓
在回复中包含工具调用标记
    ↓
后端检测工具调用
    ↓
执行工具（创建文件、分析数据等）
    ↓
将工具结果返回给虚拟人
    ↓
虚拟人整理结果
    ↓
向皇帝汇报完成情况
    ↓
皇帝可以在文件库下载文件
```

## 🔧 技术实现

### 工具调用格式
```typescript
[TOOL_CALL]
{
  "tool": "工具名称",
  "parameters": {
    "参数名": "参数值"
  }
}
[/TOOL_CALL]
```

### 工具定义
```typescript
interface Tool {
  name: string;
  description: string;
  parameters: Record<string, any>;
  execute: (params: any) => Promise<any>;
}
```

### 工具注册
```typescript
this.registerTool({
  name: 'createDocument',
  description: '创建 Markdown 格式的文档',
  parameters: {
    title: { type: 'string', description: '文档标题' },
    content: { type: 'string', description: '文档内容' }
  },
  execute: this.createDocument.bind(this)
});
```

## 💡 使用建议

### 1. 明确任务描述
在下旨时，明确说明需要"动手"的任务：
- ✅ "帮我做一个演示文稿"
- ✅ "分析这组数据"
- ✅ "创建一份文档"
- ❌ "想想怎么做"（太模糊）

### 2. 提供详细信息
给出足够的信息让 AI 决定使用哪个工具：
- ✅ "做一个包含 3 页的演示文稿，内容是..."
- ✅ "分析这组数据：[1, 2, 3, 4, 5]"
- ❌ "做点什么"（信息不足）

### 3. 检查文件库
任务完成后，到文件库查看和下载生成的文件。

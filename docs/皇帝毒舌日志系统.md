# 皇帝毒舌日志系统 - 强制回顾你的无能

## 核心理念

**记录皇帝对大臣的每一个操作，用毒舌的语言描述，大臣进游戏时必须看完才能继续**

- 每个操作都被记录下来
- 用讽刺和毒舌的语言描述
- 大臣进游戏时强制查看
- 无法跳过，必须点击确认
- 下次进游戏从上次确认之后开始

## 毒舌日志的语言风格

### 原则

1. **讽刺** - 用讽刺的语言描述
2. **毒舌** - 用毒舌的语言嘲笑
3. **夸张** - 用夸张的语言渲染
4. **拟人** - 用拟人的语言描述大臣的反应

### 例子

```
不好的例子：
"14:30 皇帝给大臣A分配了任务"
"14:35 大臣A的任务失败了"
"14:40 大臣A的排名下降了"

好的例子：
"14:30 皇帝让大臣A去唱歌，大臣A连屁都不敢放，乖乖接受了"
"14:35 大臣A唱歌失败了，30分的成绩，皇帝看了一眼就摇头了"
"14:40 大臣A的排名从第5位下降到第4位，大臣A的脸都绿了"
```

## 数据库设计

### 皇帝毒舌日志表

```sql
CREATE TABLE emperor_sarcasm_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  court_id UUID REFERENCES courts(id),
  user_id UUID REFERENCES users(id),  -- 大臣的 ID
  
  -- 日志内容
  log_type VARCHAR(50),  -- task_assigned, task_failed, task_success, ranking_changed, emperor_mocking, punishment, reward, comparison, share, grudge_changed
  log_message TEXT,      -- 毒舌日志的完整文本
  
  -- 相关数据
  related_data JSONB,    -- 相关的数据（任务ID、排名、怨气值等）
  
  -- 大臣是否已查看
  viewed_by_minister BOOLEAN DEFAULT FALSE,
  viewed_at TIMESTAMPTZ,
  
  -- 时间戳
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 大臣的日志查看进度
CREATE TABLE minister_log_progress (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  court_id UUID REFERENCES courts(id),
  user_id UUID REFERENCES users(id),
  
  -- 最后查看的日志 ID
  last_viewed_log_id UUID REFERENCES emperor_sarcasm_logs(id),
  
  -- 最后查看的时间
  last_viewed_at TIMESTAMPTZ,
  
  -- 未查看的日志数量
  unviewed_count INTEGER DEFAULT 0,
  
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

```typescript
const logRankingChange = async (courtId: string, ministerId: string, oldRank: number, newRank: number, oldScore: number, newScore: number) => {
  const minister = await getMinister(ministerId);
  
  let rankingMessage = '';
  if (newRank > oldRank) {
    rankingMessage = `${minister.name}的排名从第${oldRank}位下降到第${newRank}位，无能指数从${oldScore.toFixed(1)}升到${newScore.toFixed(1)}`;
  } else {
    rankingMessage = `${minister.name}的排名从第${oldRank}位上升到第${newRank}位，无能指数从${oldScore.toFixed(1)}降到${newScore.toFixed(1)}`;
  }
  
  const rankingReactions = [
    `${minister.name}的脸都绿了`,
    `${minister.name}的脸都红了`,
    `${minister.name}的心都凉了`,
    `${minister.name}开始后悔了`,
    `${minister.name}开始绝望了`
  ];
  
  const reaction = rankingReactions[Math.floor(Math.random() * rankingReactions.length)];
  
  await createLog({
    courtId,
    userId: ministerId,
    logType: 'ranking_changed',
    logMessage: `[${getCurrentTime()}] ${rankingMessage}，${reaction}`,
    relatedData: { oldRank, newRank, oldScore, newScore }
  });
};
```

### 3. 日志滚动显示

```typescript
const LogModal = ({ logs, onConfirm }) => {
  const [currentIndex, setCurrentIndex] = useState(0);
  
  const handleNext = () => {
    if (currentIndex < logs.length - 1) {
      setCurrentIndex(currentIndex + 1);
    }
  };
  
  const handlePrev = () => {
    if (currentIndex > 0) {
      setCurrentIndex(currentIndex - 1);
    }
  };
  
  const handleConfirm = async () => {
    // 标记所有日志为已查看
    await api.post(`/logs/mark-viewed`, {
      courtId,
      userId,
      logIds: logs.map(log => log.id)
    });
    
    onConfirm();
  };
  
  return (
    <Modal>
      <div className="log-content">
        <div className="log-item">
          {logs[currentIndex].logMessage}
        </div>
      </div>
      
      <div className="log-navigation">
        <button onClick={handlePrev} disabled={currentIndex === 0}>
          上一条
        </button>
        <span>{currentIndex + 1} / {logs.length}</span>
        <button onClick={handleNext} disabled={currentIndex === logs.length - 1}>
          下一条
        </button>
      </div>
      
      <div className="log-actions">
        <button onClick={handleConfirm} className="confirm-btn">
          确认 ({logs.length} 条日志已全部查看)
        </button>
      </div>
    </Modal>
  );
};
```

```typescript
  logType: string,
```

### 2. 获取未查看日志

```typescript
GET /api/logs/unviewed/:courtId/:userId

Response:
{
  unviewedLogs: [
    {
      id: string,
      logMessage: string,
      createdAt: timestamp
    }
  ],
  unviewedCount: number
}
```

### 3. 标记日志为已查看

```typescript
POST /api/logs/mark-viewed
{
  courtId: string,
  userId: string,
  logIds: string[]
}
```

### 4. 获取日志历史

```typescript
GET /api/logs/history/:courtId/:userId?limit=50&offset=0
```

## 这个系统的优势

1. **强制回顾** - 大臣必须看完所有日志才能继续游戏
2. **毒舌嘲笑** - 用讽刺和毒舌的语言描述，增强气人效果
3. **持久记录** - 所有操作都被记录下来，无法逃避
4. **怨气值驱动** - 看完日志后怨气值增加，推动游戏循环
5. **游戏沉浸感** - 让大臣真正感到被皇帝欺负

## 部署步骤

### 1. 提交代码到 Git

```bash
# 查看修改的文件
git status

# 添加所有修改
git add .

# 提交
git commit -m "feat: 添加召唤分身功能

- 后端: 添加获取可用人机列表和召唤分身 API
- 前端: 添加召唤分身按钮和弹窗
- 随机分配三省六部

"
```

### 2. 推送到远程仓库

```bash
# 推送到主分支
git push origin main

# 或者推送到其他分支
git push origin <your-branch-name>
```

### 3. Railway 自动部署

Railway 会自动检测到代码变更并开始部署：

1. 登录 Railway Dashboard: https://railway.app
2. 找到你的项目
3. 查看 Deployments 标签
4. 等待部署完成（通常需要 2-5 分钟）

### 4. 验证部署

部署完成后，测试新的 API：

```bash
# 测试获取可用人机列表
curl "https://backend-production-a216.up.railway.app/api/courts/YOUR_COURT_ID/available-bots?current_user_id=YOUR_USER_ID"

# 应该返回：
# {
#   "code": 0,
#   "data": [...]
# }
```

### 5. 前端测试

1. 刷新前端页面
2. 点击左上角"召唤分身"按钮
3. 应该能看到可召唤的分身列表
4. 点击任意分身进行召唤

## 常见问题

### Q: 推送后 Railway 没有自动部署？

A: 检查以下几点：
1. Railway 项目是否连接了正确的 Git 仓库
2. 是否推送到了正确的分支
3. Railway 项目设置中是否启用了自动部署

### Q: 部署失败怎么办？

A: 查看 Railway 的部署日志：
1. 进入 Railway Dashboard
2. 点击项目
3. 查看 Deployments 标签
4. 点击失败的部署查看详细日志

### Q: 部署成功但 API 还是 404？

A: 可能的原因：
1. 缓存问题 - 等待几分钟或强制刷新
2. 路由注册问题 - 检查 `backend/src/index.ts`
3. 构建问题 - 检查 `backend/dist/routes/courts.js` 是否存在

### Q: 前端还是报错？

A: 清除浏览器缓存：
1. 打开开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

## 🎯 核心功能

### 1. 日志创建
系统自动为以下操作创建讽刺日志：
- 任务分配
- 任务失败
- 任务成功
- 排名变化
- 皇帝嘲笑
- 惩罚执行
- 奖励发放

### 2. 日志显示
- 大臣登录时自动显示未查看日志
- 一次显示一条日志
- 支持上一条/下一条导航
- 显示进度 (X / Y)
- 必须查看所有日志才能继续游戏

### 3. 日志管理
- 标记日志为已查看
- 获取日志历史
- 获取日志进度
- 获取未查看日志数量

## 🚀 使用方式

### 后端调用

```typescript
import { logTaskAssignment, logTaskFailure, logTaskSuccess } from '../services/sarcasm-logs.service';

// 任务分配时
await logTaskAssignment(courtId, ministerId, ministerName, taskName, taskId);

// 任务失败时
await logTaskFailure(courtId, ministerId, ministerName, taskName, taskId, score);

// 任务成功时
await logTaskSuccess(courtId, ministerId, ministerName, taskName, taskId, score);
```

### 前端显示

```typescript
// 自动显示（在 Home.tsx 中已实现）
// 大臣登录时，如果有未查看日志，会自动显示 LogModal

// 手动检查
const response = await logsAPI.getUnviewed(courtId, userId);
const unviewedLogs = response.data.unviewedLogs;
```

## 📊 API 端点

| 方法 | 端点 | 功能 |
|------|------|------|
| POST | `/api/logs/create` | 创建日志 |
| GET | `/api/logs/unviewed/:courtId/:userId` | 获取未查看日志 |
| GET | `/api/logs/history/:courtId/:userId` | 获取日志历史 |
| POST | `/api/logs/mark-viewed` | 标记日志为已查看 |
| GET | `/api/logs/progress/:courtId/:userId` | 获取日志进度 |
| GET | `/api/logs/unviewed-count/:courtId/:userId` | 获取未查看日志数量 |

# 工具调用清理修复

## 问题描述

AI 在调用 MCP 工具或内置工具时，工具调用的技术细节（如 `[MCP_CALL]` 标记、JSON 代码等）会暴露到前端显示给用户。

## 问题原因

1. **后端流程问题**：虽然 `executeTaskWithTools` 方法会清理工具调用标记，但在某些情况下，未清理的原始回复可能被保存到数据库或广播到前端
2. **缺少前端保护**：前端没有对接收到的内容进行清理，如果后端清理失败，工具调用标记会直接显示

## 解决方案

### 后端修复

1. **优化 executeTaskWithTools 方法**（`backend/src/services/secondme.service.ts`）
   - 添加详细的日志，标记每个步骤
   - 在返回前再次清理，确保万无一失
   - 返回 `trim()` 后的内容

2. **确保所有保存和广播都使用清理后的内容**（`backend/src/routes/tasks.ts`）
   - 任务创建时：使用 `executeTaskWithTools` 返回的清理后内容
   - 准奏时：使用 `executeTaskWithTools` 返回的清理后内容
   - 驳回时：使用 `executeTaskWithTools` 返回的清理后内容
   - 所有数据库保存和 WebSocket 广播都使用清理后的 `updatedTask`

### 前端防护

添加双重清理机制（`frontend/src/pages/Home.tsx`）：

1. **cleanToolCallMarkers 函数**：移除工具调用标记
   - 移除 `[TOOL_CALL]...[/TOOL_CALL]`
   - 移除 `[MCP_CALL]...[/MCP_CALL]`

2. **cleanContent 函数**：综合清理
   - 先清理工具调用标记
   - 再清理 Markdown 格式
   - 确保显示内容干净自然

3. **应用到所有显示位置**：
   - 皇帝的任务描述气泡
   - 大臣的回复气泡
   - TTS 语音生成

## 修改的文件

- `backend/src/services/secondme.service.ts` - 优化工具调用清理逻辑
- `backend/src/routes/tasks.ts` - 确保所有保存和广播使用清理后内容
- `frontend/src/pages/Home.tsx` - 添加前端清理防护

## 测试方法

1. 创建一个需要使用工具的任务（如"帮我创建一个项目计划文档"）
2. 观察大臣的回复，确认没有出现 `[TOOL_CALL]` 或 `[MCP_CALL]` 标记
3. 确认没有出现 JSON 代码块
4. 确认回复内容自然流畅，像真人对话

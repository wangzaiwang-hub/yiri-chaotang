# 快速部署指南 - 召唤分身功能

## 问题

前端报错 404，因为新的 API 还没有部署到 Railway。

## 解决方案

### 方法 1：使用自动脚本（推荐）

```bash
./deploy-summon-feature.sh
```

### 方法 2：手动部署

#### 步骤 1：添加文件

```bash
git add backend/src/routes/courts.ts
git add frontend/src/services/api.ts
git add frontend/src/pages/Home.tsx
```

#### 步骤 2：提交

```bash
git commit -m "feat: 添加召唤分身功能"
```

#### 步骤 3：推送

```bash
git push origin main
```

#### 步骤 4：等待部署

1. 访问 Railway Dashboard: https://railway.app
2. 找到你的项目
3. 查看 Deployments 标签
4. 等待部署完成（2-5 分钟）

#### 步骤 5：测试

1. 刷新前端页面（强制刷新：Cmd+Shift+R）
2. 点击左上角"召唤分身"按钮
3. 应该能看到分身列表

## 验证部署成功

### 测试 API

```bash
# 替换 YOUR_COURT_ID 和 YOUR_USER_ID
curl "https://backend-production-a216.up.railway.app/api/courts/YOUR_COURT_ID/available-bots?current_user_id=YOUR_USER_ID"
```

应该返回：
```json
{
  "code": 0,
  "data": [...]
}
```

### 前端测试

1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 点击"召唤分身"按钮
4. 查看 `available-bots` 请求
5. 状态码应该是 200（不是 404）

## 如果还是 404

### 检查 1：确认代码已推送

```bash
git log --oneline -1
```

应该看到你的提交信息。

### 检查 2：确认 Railway 已部署

1. 登录 Railway
2. 查看最新的 Deployment
3. 状态应该是 "Success"

### 检查 3：清除缓存

浏览器强制刷新：
- Mac: `Cmd + Shift + R`
- Windows: `Ctrl + Shift + R`

### 检查 4：查看 Railway 日志

1. Railway Dashboard
2. 点击项目
3. 查看 Logs 标签
4. 搜索 "available-bots"

## 常见问题

**Q: 推送后多久能生效？**
A: Railway 通常需要 2-5 分钟完成部署。

**Q: 如何知道部署完成了？**
A: 在 Railway Dashboard 的 Deployments 标签中，最新的部署状态变为 "Success"。

**Q: 部署失败怎么办？**
A: 查看 Railway 的部署日志，通常会显示具体的错误信息。

**Q: 前端需要重新部署吗？**
A: 如果前端也有修改（Home.tsx, api.ts），需要推送前端代码。Vercel 会自动部署。

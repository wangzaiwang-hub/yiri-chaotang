# 快速部署指南

> 5 分钟内将"一日朝堂"部署到线上

## 前提条件

- GitHub 账号（代码已推送）
- Railway 账号（免费）
- Vercel 账号（免费）
- Supabase 项目（已配置）
- SecondMe API 密钥（已获取）

## 第一步：部署后端（2分钟）

### 使用 Railway 网页部署

1. 访问 https://railway.app
2. 点击 "New Project" → "Deploy from GitHub repo"
3. 选择你的仓库
4. 在 Settings 中设置：
   - Root Directory: `backend`
5. 在 Variables 中添加环境变量：
   ```
   NODE_ENV=production
   PORT=3001
   SUPABASE_URL=你的值
   SUPABASE_ANON_KEY=你的值
   SUPABASE_SERVICE_ROLE_KEY=你的值
   SECONDME_CLIENT_ID=你的值
   SECONDME_CLIENT_SECRET=你的值
   SECONDME_API_KEY=你的值
   BASE_URL=https://xxx.up.railway.app
   FRONTEND_URL=https://xxx.vercel.app
   ```
6. 点击 Deploy
7. 记录生成的域名（如：`https://xxx.up.railway.app`）

## 第二步：部署前端（2分钟）

### 使用 Vercel 网页部署

1. 访问 https://vercel.com/new
2. 导入你的 GitHub 仓库
3. 配置：
   - Root Directory: `frontend`
   - Framework Preset: `Vite`
4. 在 Environment Variables 中添加：
   ```
   VITE_API_URL=https://你的Railway域名.up.railway.app
   ```
5. 点击 Deploy
6. 记录生成的域名（如：`https://xxx.vercel.app`）

## 第三步：更新配置（1分钟）

### 3.1 更新 Railway 环境变量

回到 Railway 项目，更新以下变量：
- `BASE_URL`: 改为实际的 Railway 域名
- `FRONTEND_URL`: 改为实际的 Vercel 域名

### 3.2 更新 SecondMe 回调 URL

访问 SecondMe 开发者平台，更新回调 URL：
```
https://你的Railway域名.up.railway.app/api/auth/callback
```

### 3.3 更新 Supabase 配置

在 Supabase 项目设置 → Authentication → URL Configuration：
- Site URL: `https://你的Vercel域名.vercel.app`
- Redirect URLs: 添加 `https://你的Vercel域名.vercel.app/**`

## 第四步：测试（1分钟）

1. 访问你的前端域名
2. 点击登录
3. 完成 SecondMe 授权
4. 创建朝堂测试

## 完成！

恭喜，你的"一日朝堂"已经部署成功！

## 常见问题

### Q: 登录后跳转失败？
A: 检查 SecondMe 回调 URL 是否正确配置

### Q: 前端无法连接后端？
A: 检查 `VITE_API_URL` 环境变量是否正确

### Q: WebSocket 连接失败？
A: 确认后端部署在 Railway（不是 Vercel）

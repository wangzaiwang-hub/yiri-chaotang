# 惩罚功能完成

## 功能说明

皇帝可以惩罚表现不佳的大臣，让大臣的虚拟分身在 SecondMe Plaza 广场发布羞耻言论，并增加怨气值。

## 实现细节

### 1. 后端实现

#### API 接口
- **路径**: `POST /api/tasks/:id/punish`
- **参数**: `punishment_content` - 惩罚内容（羞耻言论）
- **功能**:
  1. 获取任务和大臣信息
  2. 使用大臣的 access_token 调用 Plaza API
  3. 以大臣的虚拟分身名义发布帖子
  4. 增加大臣 30 点怨气值（social_death 类型）
  5. 记录怨气来源

#### SecondMe Plaza API
- **接口**: `POST /api/secondme/plaza/posts/create`
- **参数**:
  - `content`: 帖子内容
  - `type`: "public"
  - `contentType`: "discussion"
- **权限**: 需要 `plaza.write` 权限
- **准入**: 需要先激活 Plaza 准入（居民证）

#### 错误处理
- 如果大臣未激活 Plaza 准入，返回 403 错误
- 前端显示提示："该大臣尚未激活 Plaza 准入"

### 3. 功能流程

```
1. 皇帝查看大臣回复
2. 点击"罚"按钮
3. 弹出惩罚弹窗
4. 输入羞耻言论内容
5. 点击"执行惩罚"
6. 后端调用 Plaza API
7. 以大臣名义发布帖子
8. 增加大臣怨气值 +30
9. 显示成功提示
```

### 4. 怨气值影响

- **惩罚类型**: social_death（社死）
- **怨气增加**: 30 点
- **记录**: 记录在 grudge_records 表中
- **来源**: caused_by_id 为皇帝 ID
- **描述**: "遭受虚拟惩罚：social_death"

## 注意事项

1. **Plaza 准入**
   - 大臣必须先激活 Plaza 准入才能发布帖子
   - 如果未激活，会返回错误提示
   - 用户需要先获取邀请码激活 Plaza

2. **权限要求**
   - 需要 `plaza.write` 权限
   - 使用大臣的 access_token 调用 API

3. **怨气值**
   - 每次惩罚增加 30 点怨气
   - 怨气值过高可能导致大臣"造反"
   - 需要平衡惩罚和怨气管理

4. **内容审核**
   - Plaza 可能有内容审核机制
   - 避免发布违规内容
   - 建议使用古风、幽默的羞耻言论

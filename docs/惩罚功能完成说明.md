# 惩罚功能完成说明

## 完成时间
2026-04-06

## 功能概述
实现了皇帝惩罚大臣的功能，皇帝输入惩罚任务描述，AI 自动生成古风羞耻文案，并以大臣名义发布到 SecondMe Plaza 广场。

## 实现细节

### 1. 后端实现 (backend/src/routes/tasks.ts)
- 接口：`POST /api/tasks/punish-minister`
- 参数：
  - `court_id`: 朝堂 ID
  - `minister_id`: 大臣 ID
  - `emperor_id`: 皇帝 ID
  - `punishment_task`: 惩罚任务描述（皇帝输入）

- 流程：
  1. 获取大臣的 SecondMe access token
  2. 使用 AI 根据惩罚任务生成古风羞耻文案（100-200字）
  3. 调用 Plaza API 以大臣名义发布帖子
  4. 增加大臣 30 点怨气值（social_death 类型）

### 2. 前端实现

#### API 服务 (frontend/src/services/api.ts)
- 方法：`taskAPI.punishMinister(courtId, ministerId, emperorId, punishmentTask)`
- 参数名已更新为 `punishment_task`（与后端一致）

#### 用户界面 (frontend/src/pages/Home.tsx)
- 惩罚入口：点击怨气值区域的大臣头像/名字
- 惩罚弹窗：
  - 标题：惩罚大臣
  - 显示惩罚对象名字
  - 输入框：皇帝描述惩罚原因
  - 占位符：`例如：办事不力，屡次拖延，辜负圣恩...`
  - 提示：AI 将根据描述自动生成羞耻文案
  - 按钮：取消 / 执行惩罚

### 3. AI 生成文案
- 风格：古风文言文
- 内容：深刻自责、愧疚、请求原谅
- 字数：100-200 字
- 语气：诚恳、卑微、羞愧

## 使用方法

### 皇帝视角
1. 在朝堂页面左下角查看怨气值区域
2. 点击大臣的头像或名字
3. 在弹窗中输入惩罚原因（例如：办事不力，屡次拖延）
4. 点击"执行惩罚"
5. AI 自动生成羞耻文案并发布到 Plaza

### 效果
- 大臣的虚拟分身在 Plaza 发布自我检讨
- 大臣怨气值增加 30 点
- 皇帝收到成功提示

## 注意事项
1. 大臣必须已激活 Plaza 准入才能发布帖子
2. 如果大臣未激活 Plaza，会提示"该大臣尚未激活 Plaza 准入"
3. 惩罚会增加怨气值，怨气值过高可能影响大臣工作积极性

## 测试建议
1. 选择一个已激活 Plaza 的大臣进行测试
2. 输入简短的惩罚原因（如：办事不力）
3. 检查 Plaza 是否成功发布帖子
4. 确认大臣怨气值是否增加

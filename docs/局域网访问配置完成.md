## 常见问题

### Q: 局域网其他电脑无法访问？

A: 检查防火墙设置：

**macOS:**
```bash
# 允许端口 3000 和 3001
sudo pfctl -d  # 临时禁用防火墙测试
```

或者在系统偏好设置 → 安全性与隐私 → 防火墙 → 防火墙选项中添加例外。

**Windows:**
```bash
# 允许端口
netsh advfirewall firewall add rule name="Node 3000" dir=in action=allow protocol=TCP localport=3000
netsh advfirewall firewall add rule name="Node 3001" dir=in action=allow protocol=TCP localport=3001
```

### Q: WebSocket 连接失败？

A: 确认：
1. 后端正在运行
2. 使用 `ws://` 而不是 `wss://`
3. 防火墙允许 3001 端口

### Q: 如何查看本机局域网 IP？

**macOS/Linux:**
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

**Windows:**
```bash
ipconfig | findstr IPv4
```

### Q: IP 地址变了怎么办？

如果路由器重新分配了 IP 地址，需要：
1. 查看新的 IP 地址
2. 修改 `frontend/src/services/api.ts` 中的 `API_BASE_URL`
3. 修改 `frontend/src/pages/Home.tsx` 中的 `wsUrl`
4. 重启前端和后端

## 使用环境变量（推荐）

为了方便切换环境，建议使用环境变量：

**`frontend/.env.local`**（本地开发）
```
VITE_API_URL=http://192.168.124.47:3001
VITE_WS_URL=ws://192.168.124.47:3001
```

**`frontend/.env.production`**（生产环境）
```
VITE_API_URL=https://backend-production-a216.up.railway.app
VITE_WS_URL=wss://backend-production-a216.up.railway.app
```

然后在代码中使用：
```typescript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://192.168.124.47:3001';
const wsUrl = import.meta.env.VITE_WS_URL || 'ws://192.168.124.47:3001';
```

这样就不需要每次手动修改代码了！

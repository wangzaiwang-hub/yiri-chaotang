## 迁移内容

### 创建的表

#### 1. `emperor_sarcasm_logs` - 皇帝毒舌日志表

```sql
CREATE TABLE emperor_sarcasm_logs (
  id UUID PRIMARY KEY,
  court_id UUID NOT NULL,
  user_id UUID NOT NULL,
  log_type VARCHAR(50) NOT NULL,
  log_message TEXT NOT NULL,
  related_data JSONB,
  viewed_by_minister BOOLEAN DEFAULT FALSE,
  viewed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**字段说明：**
- `id`: 日志唯一标识
- `court_id`: 朝堂 ID
- `user_id`: 大臣用户 ID
- `log_type`: 日志类型（task_assigned, task_failed, task_success, ranking_changed, emperor_mocking, punishment, reward, comparison, share, grudge_changed）
- `log_message`: 毒舌日志文本
- `related_data`: 相关数据（JSON 格式）
- `viewed_by_minister`: 大臣是否已查看
- `viewed_at`: 查看时间
- `created_at`: 创建时间

#### 2. `minister_log_progress` - 大臣日志查看进度表

```sql
CREATE TABLE minister_log_progress (
  id UUID PRIMARY KEY,
  court_id UUID NOT NULL,
  user_id UUID NOT NULL,
  last_viewed_log_id UUID,
  last_viewed_at TIMESTAMPTZ,
  unviewed_count INTEGER DEFAULT 0,
  updated_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ
);
```

**字段说明：**
- `id`: 记录唯一标识
- `court_id`: 朝堂 ID
- `user_id`: 大臣用户 ID
- `last_viewed_log_id`: 最后查看的日志 ID
- `last_viewed_at`: 最后查看的时间
- `unviewed_count`: 未查看的日志数量
- `updated_at`: 更新时间
- `created_at`: 创建时间

### 创建的函数

#### 1. `update_unviewed_count()` - 更新未查看日志计数

当新增日志时自动调用，更新 `minister_log_progress` 表中的 `unviewed_count`。

#### 2. `mark_logs_as_viewed(p_court_id, p_user_id, p_log_ids)` - 标记日志为已查看

```sql
SELECT * FROM mark_logs_as_viewed(
  'court-id-here',
  'user-id-here',
  ARRAY['log-id-1', 'log-id-2']
);
```

#### 3. `get_unviewed_logs(p_court_id, p_user_id, p_limit)` - 获取未查看的日志

```sql
SELECT * FROM get_unviewed_logs(
  'court-id-here',
  'user-id-here',
  50
);
```

#### 4. `get_log_history(p_court_id, p_user_id, p_limit, p_offset)` - 获取日志历史

```sql
SELECT * FROM get_log_history(
  'court-id-here',
  'user-id-here',
  50,
  0
);
```

#### 5. `create_sarcasm_log(p_court_id, p_user_id, p_log_type, p_log_message, p_related_data)` - 创建日志

```sql
SELECT * FROM create_sarcasm_log(
  'court-id-here',
  'user-id-here',
  'task_assigned',
  '[14:30] 皇帝让大臣A去唱歌，大臣A连屁都不敢放，乖乖接受了',
  '{"taskId": "task-id", "taskName": "唱歌"}'::JSONB
);
```

#### 6. `log_grudge_change()` - 记录怨气值变化

当 `grudge_values` 表更新时自动调用，创建相应的日志。

### 创建的索引

```sql
CREATE INDEX idx_emperor_logs_court_user ON emperor_sarcasm_logs(court_id, user_id);
CREATE INDEX idx_emperor_logs_created_at ON emperor_sarcasm_logs(created_at);
CREATE INDEX idx_emperor_logs_viewed ON emperor_sarcasm_logs(viewed_by_minister);
CREATE INDEX idx_minister_progress_court_user ON minister_log_progress(court_id, user_id);
```

### 创建的视图

#### `unviewed_logs_view` - 未查看日志视图

```sql
SELECT * FROM unviewed_logs_view;
```

## 常见问题

### Q: 迁移失败，提示权限不足？

A: 确保使用的是 `SUPABASE_SERVICE_KEY` 而不是 `SUPABASE_ANON_KEY`。Service Key 有更高的权限。

### Q: 如何回滚迁移？

A: 在 Supabase 控制台执行以下 SQL：

```sql
-- 删除触发器
DROP TRIGGER IF EXISTS trigger_update_unviewed_count ON emperor_sarcasm_logs;

-- 删除函数
DROP FUNCTION IF EXISTS update_unviewed_count();
DROP FUNCTION IF EXISTS mark_logs_as_viewed(UUID, UUID, UUID[]);
DROP FUNCTION IF EXISTS get_unviewed_logs(UUID, UUID, INT);
DROP FUNCTION IF EXISTS get_log_history(UUID, UUID, INT, INT);
DROP FUNCTION IF EXISTS create_sarcasm_log(UUID, UUID, VARCHAR, TEXT, JSONB);
DROP FUNCTION IF EXISTS log_grudge_change();

-- 删除视图
DROP VIEW IF EXISTS unviewed_logs_view;

-- 删除索引
DROP INDEX IF EXISTS idx_emperor_logs_court_user;
DROP INDEX IF EXISTS idx_emperor_logs_created_at;
DROP INDEX IF EXISTS idx_emperor_logs_viewed;
DROP INDEX IF EXISTS idx_minister_progress_court_user;

-- 删除表
DROP TABLE IF EXISTS minister_log_progress;
DROP TABLE IF EXISTS emperor_sarcasm_logs;
```

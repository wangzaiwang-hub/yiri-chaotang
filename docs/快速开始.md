# 快速开始指南

## 第一步：安装依赖

```bash
# 安装根目录依赖（用于并发运行）
npm install

# 安装所有项目依赖
npm run install:all
```

## 第二步：配置 Supabase

1. 访问 https://supabase.com 并创建新项目
2. 进入 SQL Editor
3. 复制 `database/init.sql` 的内容并执行
4. 进入 Settings > API，复制以下信息：
   - Project URL
   - anon public key
   - service_role key (保密)

## 第三步：配置 SecondMe

1. 访问 https://develop.second-me.cn/integrations/list
2. 点击"创建应用"
3. 填写应用信息：
   - 应用名称：朝堂
   - 回调 URL：`http://localhost:3001/api/auth/callback`
   - 权限范围：`userinfo chat.write chat.read`
4. 创建后复制：
   - Client ID
   - Client Secret

## 第四步：配置后端环境变量

```bash
cd backend
cp .env.example .env
```

编辑 `backend/.env`：

```env
NODE_ENV=development
PORT=3001

# Supabase（从第二步获取）
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_KEY=your-service-key

# SecondMe（从第三步获取）
SECONDME_CLIENT_ID=your-client-id
SECONDME_CLIENT_SECRET=your-client-secret
BASE_URL=http://localhost:3001
FRONTEND_URL=http://localhost:3000

# OpenAI（可选，用于生成虚拟惩罚内容）
OPENAI_API_KEY=sk-your-openai-key

# JWT
JWT_SECRET=your-random-secret-string
```

## 第五步：启动开发服务器

```bash
# 在项目根目录运行（会同时启动前后端）
npm run dev
```

或者分别启动：

```bash
# 终端 1：启动后端
cd backend
npm run dev

# 终端 2：启动前端
cd frontend
npm run dev
```

## 第六步：访问应用

打开浏览器访问：http://localhost:3000

## 测试流程

1. 点击"使用 SecondMe 登录"
2. 在 SecondMe 授权页面登录并授权
3. 授权成功后会自动跳转回应用
4. 创建一个朝堂
5. 邀请好友加入（需要好友也授权 SecondMe）
6. 开始发布任务，体验游戏！

## 常见问题

### Q: SecondMe 授权后跳转失败？
A: 检查 `backend/.env` 中的 `BASE_URL` 和 `FRONTEND_URL` 是否正确

### Q: 数据库连接失败？
A: 检查 Supabase 配置是否正确，确保使用的是 service_role key

### Q: 任务执行失败？
A: 确保用户已经通过 SecondMe OAuth2 授权，且 Token 未过期

### Q: 定时任务不执行？
A: 开发环境下定时任务默认不启动，可以在 `backend/src/index.ts` 中手动启动

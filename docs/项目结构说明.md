## 核心模块说明

### 后端模块

#### 1. 认证模块 (auth.ts)
- SecondMe OAuth2 授权流程
- Token 管理（获取、刷新）
- 用户信息获取

#### 2. 朝堂管理 (courts.ts)
- 创建朝堂
- 邀请成员
- 查看成员列表
- 怨气排行榜

#### 3. 任务系统 (tasks.ts)
- 发布任务
- 虚拟分身执行任务（SSE 流式）
- 任务审批（通过/打回）
- 怨气值累计

#### 4. 虚拟朋友圈 (moments.ts)
- 创建虚拟朋友圈
- 查看朋友圈列表
- 点赞功能

#### 5. 怨气系统 (grudge.ts)
- 怨气记录查询
- 怨气排行榜
- 复仇清单

#### 6. 每日结算 (daily-settlement.ts)
- 定时任务（每日 00:00）
- 计算怨气排行
- 皇帝轮换
- 重置怨气值

### 前端模块

#### 1. 登录页 (Login.tsx)
- SecondMe OAuth2 授权
- 处理回调
- 保存用户信息

#### 2. 朝堂首页 (Home.tsx)
- 显示当前皇帝
- 今日圣旨列表
- 怨气排行榜
- 复仇倒计时

#### 3. 帝王宝座 (Throne.tsx)
- 发布任务表单
- 选择臣子
- 设置任务类型和截止时间

#### 4. 虚拟朋友圈 (Moments.tsx)
- 显示虚拟惩罚内容
- 点赞功能
- 过期时间倒计时

## 数据流

### 1. 用户登录流程
```
用户点击登录 
→ 跳转到 SecondMe 授权页 
→ 用户授权 
→ 回调到后端 
→ 换取 Access Token 
→ 保存用户信息 
→ 跳转到前端首页
```

### 2. 任务执行流程
```
皇帝发布任务 
→ 保存到数据库 
→ 臣子虚拟分身执行 
→ 调用 SecondMe Chat API 
→ SSE 流式返回结果 
→ 保存任务结果 
→ 增加怨气值
```

### 3. 每日结算流程
```
定时任务触发（00:00） 
→ 查询所有朝堂 
→ 计算怨气排行 
→ 怨气最高者登基 
→ 更新皇帝 
→ 重置怨气值 
→ 记录登基历史
```

## 技术要点

### 1. SSE 流式响应
- 后端使用 `res.setHeader('Content-Type', 'text/event-stream')`
- 前端使用 `EventSource` API 接收流式数据
- 实现打字机效果的实时显示

### 2. OAuth2 认证
- 标准授权码流程
- State 参数防 CSRF
- Token 自动刷新机制

### 3. 定时任务
- 使用 node-cron
- 指定时区（Asia/Shanghai）
- 自动结算和皇帝轮换

### 4. 状态管理
- Zustand 轻量级状态管理
- 持久化存储（localStorage）
- React Query 数据缓存

## 开发建议

### 1. 本地开发
- 使用 `npm run dev` 同时启动前后端
- 后端端口：3001
- 前端端口：3000
- 前端代理 `/api` 到后端

### 2. 调试技巧
- 查看后端日志：`backend/combined.log`
- 使用 Chrome DevTools 查看 SSE 流
- Supabase Dashboard 查看数据库

### 3. 测试流程
1. 先测试 OAuth2 登录
2. 创建朝堂并邀请成员
3. 发布任务测试虚拟分身
4. 查看怨气值累计
5. 手动触发结算测试

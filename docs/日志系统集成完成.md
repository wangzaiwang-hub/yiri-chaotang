# 日志系统集成完成

## 已完成的工作

### 1. 后端集成

在 `backend/src/routes/tasks.ts` 中添加了日志创建调用：

#### 任务创建时（POST /api/tasks）
```typescript
// 创建任务分配日志
await sarcasmLogsService.logTaskAssignment(
  court_id,
  assignee_id,
  ministerName,
  title,
  task.id
);
```

#### 任务批准时（POST /api/tasks/:id/approve）
```typescript
// 创建任务成功日志（准奏）
await sarcasmLogsService.logTaskSuccess(
  task.court_id,
  task.assignee_id,
  ministerName,
  task.title,
  task.id,
  85 // 准奏默认给 85 分
);
```

#### 任务驳回时（POST /api/tasks/:id/reject）
```typescript
// 创建任务失败日志（驳回）
await sarcasmLogsService.logTaskFailure(
  task.court_id,
  task.assignee_id,
  ministerName,
  task.title,
  task.id,
  40 // 驳回默认给 40 分
);
```

#### 惩罚大臣时（POST /api/tasks/punish-minister）
```typescript
// 创建惩罚日志
await sarcasmLogsService.logPunishment(
  court_id,
  minister_id,
  ministerNickname,
  'public' // 公开羞辱类型
);
```

## 如何测试

### 方法 1: 手动游戏测试（推荐）

1. **登录游戏作为皇帝**
   - 打开游戏，使用皇帝账号登录

2. **创建任务**
   - 在朝堂页面创建一个新任务
   - 分配给某个大臣
   - 这会触发 `logTaskAssignment()` 创建日志

3. **等待大臣回复**
   - 大臣的虚拟人会自动回复

4. **批准或驳回任务**
   - 点击"准奏"按钮 → 触发 `logTaskSuccess()`
   - 或点击"驳回"按钮 → 触发 `logTaskFailure()`

5. **查看日志**
   - 刷新大臣的页面
   - 应该会弹出日志弹窗
   - 显示刚才创建的日志

### 方法 2: API 测试

运行测试脚本：
```bash
cd backend
node test-log-integration.js
```

### 方法 3: 直接查询数据库

```sql
-- 查看最新的日志
SELECT * FROM emperor_sarcasm_logs 
WHERE court_id = 'your-court-id' 
ORDER BY created_at DESC 
LIMIT 10;

-- 查看未查看的日志
SELECT * FROM emperor_sarcasm_logs 
WHERE court_id = 'your-court-id' 
AND user_id = 'your-minister-id'
AND viewed_by_minister = false
ORDER BY created_at DESC;
```

## 日志触发点总结

| 操作 | 触发位置 | 日志类型 | 函数 |
|------|---------|---------|------|
| 创建任务 | `POST /api/tasks` | `task_assigned` | `logTaskAssignment()` |
| 批准任务 | `POST /api/tasks/:id/approve` | `task_success` | `logTaskSuccess()` |
| 驳回任务 | `POST /api/tasks/:id/reject` | `task_failed` | `logTaskFailure()` |
| 惩罚大臣 | `POST /api/tasks/punish-minister` | `punishment` | `logPunishment()` |

## 故障排查

### 问题：日志没有创建

**可能原因：**
1. 数据库函数未正确部署
2. 大臣的 `user_id` 不正确
3. 朝堂的 `court_id` 不正确

**解决方法：**
```bash
# 1. 检查数据库函数是否存在
psql -d your_database -c "\df create_sarcasm_log"

# 2. 检查日志表是否存在
psql -d your_database -c "\d emperor_sarcasm_logs"

# 3. 手动创建测试日志
node backend/test-create-logs-for-court.js
```

### 问题：日志创建了但前端不显示

**可能原因：**
1. 前端 API 调用失败
2. 日志已被标记为已查看
3. 角色判断错误（皇帝端不应该显示）

**解决方法：**
```bash
# 1. 检查前端控制台日志
# 打开浏览器开发者工具，查看 Console

# 2. 测试 API 是否正常
node backend/test-api-logs.js

# 3. 检查日志状态
# 在数据库中查询 viewed_by_minister 字段
```

### 问题：标记日志为已查看失败

**可能原因：**
1. 数据库函数 `mark_logs_as_viewed` 返回类型错误
2. 已经修复，确保使用最新的 `database/fix_mark_logs_function.sql`

**解决方法：**
```bash
# 重新执行修复脚本
psql -d your_database -f database/fix_mark_logs_function.sql
```

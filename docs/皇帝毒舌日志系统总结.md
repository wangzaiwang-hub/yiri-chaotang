# 皇帝毒舌日志系统 - 完整总结

## 🎯 核心概念

**用毒舌的语言记录皇帝对大臣的每一个操作，大臣进游戏时必须看完所有日志才能继续**

这是一个强制回顾系统，让大臣无法逃避自己的无能。

---

## 📦 已交付的内容

### 1. 数据库层

**文件：**
- `database/add_emperor_sarcasm_logs.sql` - 完整版
- `database/add_emperor_sarcasm_logs_simple.sql` - 简化版（推荐）

**包含：**
- 2 个表：`emperor_sarcasm_logs`、`minister_log_progress`
- 6 个函数：日志创建、查询、更新等
- 1 个触发器：自动更新未查看计数
- 1 个视图：未查看日志视图
- 4 个索引：性能优化

### 2. 后端服务层

**文件：** `backend/src/services/sarcasm-logs.service.ts`

**核心函数：**
- `createSarcasmLog()` - 创建日志
- `getUnviewedLogs()` - 获取未查看日志
- `getLogHistory()` - 获取日志历史
- `markLogsAsViewed()` - 标记为已查看
- `getLogProgress()` - 获取进度
- `getUnviewedLogCount()` - 获取未查看数量

**专用日志函数：**
- `logTaskAssignment()` - 任务分配
- `logTaskFailure()` - 任务失败
- `logTaskSuccess()` - 任务成功
- `logRankingChange()` - 排名变化
- `logEmperorMocking()` - 皇帝嘲笑
- `logPunishment()` - 惩罚
- `logReward()` - 奖励

### 3. 后端 API 层

**文件：** `backend/src/routes/sarcasm-logs.ts`

**API 端点：**
- `GET /api/logs/unviewed/:courtId/:userId` - 获取未查看日志
- `GET /api/logs/history/:courtId/:userId` - 获取日志历史
- `POST /api/logs/mark-viewed` - 标记为已查看
- `GET /api/logs/progress/:courtId/:userId` - 获取进度
- `GET /api/logs/unviewed-count/:courtId/:userId` - 获取未查看数量

## 💡 使用示例

### 创建任务分配日志

```typescript
import { logTaskAssignment } from './services/sarcasm-logs.service';

await logTaskAssignment(
  courtId,
  ministerId,
  '大臣A',
  '唱歌',
  taskId
);

// 生成的日志：
// [14:30] 皇帝让大臣A去唱歌，大臣A连屁都不敢放，乖乖接受了
```

### 创建排名变化日志

```typescript
import { logRankingChange } from './services/sarcasm-logs.service';

await logRankingChange(
  courtId,
  ministerId,
  '大臣A',
  5,    // 旧排名
  4,    // 新排名
  0.5,  // 旧无能指数
  1.2   // 新无能指数
);

// 生成的日志：
// [14:40] 大臣A的排名从第5位下降到第4位，无能指数从0.5升到1.2，大臣A的脸都绿了
```

### 获取未查看日志

```typescript
import { getUnviewedLogs } from './services/sarcasm-logs.service';

const logs = await getUnviewedLogs(courtId, ministerId, 50);

// 返回所有未查看的日志
```

### 标记日志为已查看

```typescript
import { markLogsAsViewed } from './services/sarcasm-logs.service';

await markLogsAsViewed(courtId, ministerId, logIds);

// 标记日志为已查看，更新进度
```

## 🎮 游戏流程集成

### 进游戏时

```
1. 检查是否有未查看日志
   ↓
2. 如果有，显示日志弹窗
   ↓
3. 大臣必须查看所有日志
   ↓
4. 点击确认后标记为已查看
   ↓
5. 怨气值增加（看完日志后）
   ↓
6. 继续游戏
```

### 游戏中的日志记录

```
任务分配 → 创建日志
   ↓
任务完成/失败 → 创建日志
   ↓
排名变化 → 创建日志
   ↓
皇帝嘲笑 → 创建日志
   ↓
惩罚/奖励 → 创建日志
```

## 🎯 这个系统的优势

### 1. 强制回顾
- 大臣必须看完所有日志才能继续
- 无法逃避自己的无能

### 2. 毒舌嘲笑
- 用讽刺的语言描述
- 增强气人效果

### 3. 持久记录
- 所有操作都被记录
- 无法改写历史

### 4. 怨气值驱动
- 看完日志后怨气值增加
- 推动游戏循环

### 5. 游戏沉浸感
- 让大臣真正感到被欺负
- 增强游戏体验
